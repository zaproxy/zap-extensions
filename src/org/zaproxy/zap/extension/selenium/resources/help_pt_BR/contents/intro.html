Selenium

O complemento Selenium fornece WebDrivers, para outros complementos, para invocar e controlar remotamente navegadores da web. Ele traz embutido o HtmlUnit, navegador web baseado em Java headless.

Há suporte para os seguintes navegadores web:
Nome ID Requisitos/notas
Chrome chrome Chrome - requer ChromeDriver, se não no caminho do sistema, pode ser definido nas opções. Para obter mais informações sobre ChromeDriver e como obtê-lo vá ao site ChromeDriver.
Firefox firefox Firefox - as versões seguintes foram testadas: 45 (ESR), 46 e 47.0.1 (versão mais antiga pode funcionar também). Sabe-se que algumas versões não funcionam, por exemplo a 47,0. As versões mais recentes (≥ 48) requerem geckodriver. Para obter mais informações sobre geckodriver e como obtê-lo consulte o site geckodriver (ver nota de rodapé para ressalva quando usando geckodriver).
HtmlUnit htmlunit Pacote de navegador, não tem qualquer exigência.
Internet Explorer ie Chrome - requer ChromeDriver, se não no caminho do sistema, pode ser definido nas opções. Para obter mais informações sobre IEDriverServer veja o site de IEDriverServer (ver nota de rodapé para ressalva ao usar o Internet Explorer).
Opera opera  
PhantomJS phantomjs PhantomJS - requer PhantomJS binário, se não no caminho do sistema, ele pode ser definido nas opções. Para obter mais informações sobre PhantomJS e como obtê-lo vá ao site de PhantomJS (ver nota de rodapé para ressalva quando usando PhantomJS).
Safari safari  

Para usar o Firefox, Chrome, Internet Explorer, Opera e Safari PhantomJS, você deve tê-los instalados em seu sistema. A ID do navegador pode ser usada para escolher o navegador ao configurar ZAP através de linha de comando ou usando a API ZAP (por exemplo, para definir a aranha de AJAX para usar um ou outro navegador).

Alguns dos requisitos (por exemplo, WebDrivers) dos navegadores podem ser configurados na tela de opções do selênio.

Nota: Complementos do ZAP podem adicionar navegadores.

Firefox/geckodriver Nota: Há um problema (Bug 1103196) que impede que sites HTTPS sejam usados nas versões ≥ 48 e 52 <.

PhantomJS Nota: Há um problema (edição #11342) que impede que sites no localhost, 127.0.0.1 e:: 1 de serem intermediados via proxy através do ZAP. Até que uma solução apareça é aconselhável não usá-lo nesses casos. Algumas extensões podem mostrar mensagens de alerta quando isso acontecer. Como uma solução alternativa se poderia definir, no arquivo hosts um mapeamento de nome de domínio para o endereço local e usar esse nome de domínio em vez disso.

Nota sobre o Internet Explorer: Nem todas as versões de Internet Explorer funcionam de cara; consulte o Site IEDriverServer para mais detalhes sobre como configurá-lo.
