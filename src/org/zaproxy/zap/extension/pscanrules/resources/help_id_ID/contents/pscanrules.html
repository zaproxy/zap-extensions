Info aplikasi

Periksa respon server HTTP 500 - Internal Server Error jenis tanggapan yang mendukung tali kesalahan.

Cache kontrol

Memeriksa ''Cache-Kontrol'' dan ''Pragma'' Header respon terhadap industri umum pengaturan praktek terbaik untuk perlindungan konten sensitif.
Diambang menengah dan tinggi dianggap hanya bebas-kesalahan atau non-redirect teks tanggapan (tidak termasuk JavaScript).
Di ambang rendah semua tanggapan dari gambar dan CSS dianggap termasuk dan pengalihan.

Jenis konten yang hilang

Menimbulkan suatu peringatan jika respon tersebut tidak memiliki jenis konten atau jika headernya ada namun nilainya kosong.

Cookie HttpOnly

Memastikan bahwa saat cookie ditetapkan, mereka diberi label HttpOnly, Flag HtppOnly menunjukkan ke pencarianbahwa cookie tersebut sedang diatur dan harus segara ditindaklanjuti oleh script sisi klien (seperti JavaScript).

Bendera aman cookie

Mencaro cookie yang di set selama sesi HTPPS, menimbulkan peringatan bagi mereka yang ditetapkan tetapi tidak menyertakan bendera aman. Cookie ditetapkan dengan bendera yang aman tidak akan dikirim selama sesi HTTP polos.

Penyisipan Script lintas Domain

Memvalidasi apakahscript disertakan dalam domain selain domain yang memhosting konten. Dengan melihat ''src'' atribut ''src'' dari tag dalam tanggapan.
Script memungkinksn Cross-Domain:
  • Setiap scrip dengan atribut '' integritas'' tidak diabaikan - nilai integritas tidak dicentang karena ini kan diperiksa oleh browser
  • Diambang menengah dan tinggi jika URL scrip yang jatuh dalam konteks yang juga mencakup URL dasar pesan peringatan tidak akan dibangkitkan.

Header XSS perlindungan

Cek untuk keberadaan dan nilai/pengaturan header X-XSS-perlindungan. Header respon ini dapat digunkan untuk mengkonfigurasi agen pengguna built-in perlindungan XSS reflektif.
Pada ambang menengah dan tinggi hanya tanggapan HTML bebas-kesalahan atau non-redirect yang dipertimbangkan.
Pada ambang batas rendah semua tanggpan teks dipertimbangkantermasuk kesalahan dan pengalihan.

Kontenb Campuran

Untuk konten yang ditayangkan melalui HTTPS Nmenganalisis semua src atribut dalam respon mencari orang-orang yang bersumber melalui HTTP polos.

Perlengkapan kata sandi otomatis

Mencari ''kata sandi'' tipe ladang input dan memeriksa pengaturan ''aotucomplete=mati''.

Pengungkapan Alamat Pribadi

Cek isi respon untuk dimasukkan alamat IPv4 1918 serta Amazon EC2 pribadi hostname (misalnya, ip-10-0-56-78). Informasi ini dapat memberikan penyerangan informasi yang berguna tentang skema alamat IP dari jaringan internal, dan mungkin bisa membantu untuk lebih lanjut serangan penargetan sitem internal.

Sidang ID di URl Menulis Kembali

Pemindai ini memeriksa adanya parameter jenis token sesi yang ditulis ulang ke URL. Untuk membantui mengurangi keslahan positif pemandaian cek panjang nilai token, jika nilai parameter tidak lebih dari 8 karakter panjangnya maka parameter akan diabaikan ( yaitu: Survei? sld = 5 akan ditandai sebagai rentan).

X-Konten-Tipe-Pilihan

Pemeriksaan pemindai ini untuk header Anti-MIME-Sniffing header X-Konten-Tipe-Pilihsn dan memastikiannya disetel ke 'nosnif'.
Di ambang menengah dan tinggi pemindai ini tidak akan waspada pada klien atau server tanggapan kesalahan atau pengalihan.
Pada ambang abtas rendah semua tanggapan teks dipertimbangkan dan pengalihan.

X-Bingkai-Pilihan Header Pemindai

Pemindai ini memeriksa keberadaan dan validitas header X-Bingkai-Pilihan.
Di amabng menengah dan tinggi ini hanaya terlihat kesalahan di bebas-kesalahan atau non-redirect HTML tanggapan.
Pada ambang batas rendah semua tanggapan teks dipertimbangkan dan pengalihan.
Kondisi tersebut dapat mengakibatkan peringatan:
  • X-Bingkaio-Pilihan Header tidak set: jika header X-Bingkai-Pilihan hilang dari respon sepenuhnya.
  • Beberapa entri X-Bingkai-Pilihan header: Bila lebih dari satu header X-Bingkai-Pilhan terdeteksi pada respon.
  • X-Bingkai-Pilihan Ditetapkan melalui META (Non-compliant dengan Spec): A ''http-equiv'' ditemukan dalam respon yang mencoba mendefenisikan X-Bingkai-Pilihan, yang tidak didukung oleh spesifikasi.
  • X-Bingkai-Pilihan pengaturan Malformed: Header hadir dengan nilai tidak ada, atau nilai adalah tidak seperti yang diharapkan (yaitu: 'Tolak''. ''SAMEORIGIN''. atau ''IZINKAN-DARI'').
Secara default lansiran tidak akan dibangkitkan dalam respon termasuk kebijakan konten keamanan 'frame-leluhur' elemen seperti ini diutamakan daripada header X-Bingkai_pilihan. Namun di ambang rendah berbagai persoalan diatas masih akan dilaporkan tetapi pada rendah resiko.
