Pagsuporta sa Bukas na Ispesisikasyon ng API

Ang add-on na ito ay magpapahintulot sa iyo na ispayder at mag-import ng OpenAPI (Swagger) na mga kahulugan (mga bersyong 1.2 at 2.0).

Ang add-on ay awtomatikong magtutuklas ng kahit anong Bukas na API na mga kahulugan at magspayder sa kanila habang sila ay nasa saklaw.

UI

2 mga item ng menu ang nadagdag sa Tools na menu:
  • Mag-import ng isang Bukas na API na kahulugan mula sa lokal na sistema ng file
  • Mag-import ng Bukas na API na kahulugan mula sa isang URL
Ang URL sa Pag-import na dialogo ay sumusuporta ng opsyonal na karagdagdagang field na magpapahintulot sa iyo na ipawalang-bisa ang Host (at ang port) sa loob ng kahulugan kasama ang isang alternatibo na iyong isinu-supply.
Ang pagpipilian ay hindi inilaan para sa pag-import ng dokumento dahil madaling magbago ng isang lokal ng dokumento na magagamit sa ibang host.

API

Ang sumusunod na mga operasyon ay nadagdag sa API:
  • ACTION importFile (file)
  • ACTION importUrl (url, hostOverride)
Ang mga kahulugan ay ma-iimport ng sabay at ang kahit anong mga babala ay ibabalik.

Linya ng Pag-utos

Ang sumusunod na Linya ng Pag-utos na mga opsyon ay nadagdag:
  • -openapifile <filename> : Mag-import ng Bukas na API na kahulugan mula sa tinukoy na pangalan ng dokumento
  • -openapiurl <url> : Mag-import ng Bukas na API na kahulugan mula sa natukoy na URL
Ang mga kahulugan ay ma-iimport ng sabay at ang kahit anong mga babala ay mapapakita sa linya ng pag-utos.

Mga Balyu na Natukoy ng Tagagamit

Ang default na mga balyu na ginamit habang nag-iimport ng Bukas na API na mga kahulugan.
Ito ay maaring mapawalang bisa gamit ang Tagahawak ng Form na add-on na magpapahintulot sa iyo na tukuyin ang mga sarili mong balyu.
Sa karamihang panahon ito ay magiging mga simpleng mga balyu (katulod ng mga string at integer) pero sa ibang mga kaso ay kailangan mong magtukoy ng naistrakturang mga balyu. 
{ "id": 0, "name": "Freda" }
