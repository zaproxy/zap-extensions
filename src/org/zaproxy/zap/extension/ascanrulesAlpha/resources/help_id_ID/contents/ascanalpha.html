Active Scan Aturan - alpha

Berikut alpha kualitas active scan aturan termasuk dalam hal ini add-on:

Apache Range Header DoS (CVE-2011-3192)

Tes untuk melihat apakah server tersebut rentan Apache Range Header Denial of Service masalah, dengan meminta sebelas (11) berbeda byte berkisar. Sebelas rentang adalah salah satu lebih dari diterima oleh server ditambal/tetap.

Detektor slack cookie

Pengujian cookies untuk mendeteksi jika beberapa tidak memiliki efek pada ukuran respons saat dihilangkan, terutama cookie yang berisi nama "sidang" atau "identitas pengguna"

Pemindai ELMAH

Pengujian untuk melihat apakah modul log dan modul Pelacakan salah (Elmah.axd) Modul HTTP tersedia. Walaupun modul ini sangat berguna untuk para pengembang dan pemangku kepentingan lainnya ini juga dapat bocor sejumlah besar informasi yang seorang analis keamanan atau individu berbahaya mungkin akan tertarik dalam.

Pemindai ELMAH berbasis Microsoft Technology: IIS, Windows, ASP, dan MSSQL.

Ekspresi Bahasa Injeksi

Software konstruksi semua atau bagian dari sebuah expression language (EL) pernyataan dalam Java Server Page (JSP) menggunakan eksternal yang dipengaruhi masukan dari hulu komponen, tetapi tidak menetralisir atau salah menetralkan unsur-unsur khusus yang bisa mengubah dimaksudkan EL pernyataan sebelum dieksekusi. Dalam versi tertentu dari Spring 3.0.5 dan sebelumnya, ada kerentanan (CVE-2011-2730) dalam bahasa ekspresi akan dievaluasi dua kali, yang secara efektif terkena aplikasi untuk EL injeksi. Namun, bahkan untuk versi yang lebih baru, kelemahan ini masih mungkin tergantung pada konfigurasi.

untuk MENDAPATKAN POSTING Scanner

Scanner ini membutuhkan waktu application/x-www-form-urlencoded POSTING permintaan, perubahan parameter dari POSTING untuk MENDAPATKAN dan resubmits permintaan. Jika MENDAPATKAN respon yang sama seperti aslinya POSTING respon maka alarm akan berbunyi. Sementara ini tidak selalu mewakili kelemahan keamanan kepada dirinya sendiri mungkin mengindikasikan bahwa serangan atau kelemahan lain dapat diperluas atau disederhanakan. (Seperti POS berdasarkan Cross-Site Scripting (XSS) serangan yang berubah.)

HTTP Satunya Situs

Ini aktif scanner memeriksa apakah sebuah situs HTTP disajikan di bawah HTTPS.

Httpoxy

Pemindai aktif ini memeriksa apakah sebuah situs menggunakan header HTTP Proxy yang ditentukan dalam permintaan.
Ini membuat proxy HTTP yang mendengarkan semua antarmuka pada port bebas yang ditetapkan secara acak. Kemudian mengirimkan serangkaian permintaan ke server target dengan header HTTP Proxy yang disetel ke masing-masing yang tersedia Alamat IP dan port yang didengarkannya. Jika permintaan diterima di port baru maka server sangat mungkin rentan.
PENTING - komputer yang dijalankan ZAP harus menerima permintaan masuk pada port yang sewenang-wenang - Jika firewall mencegah koneksi masuk maka aturan ini tidak akan berhasil.

HTTPS sebagai pemindai HTTP

Pemindai aktif ini mencoba mengakses konten yang awalnya diakses melalui HTTPS (SSL/TLS) melalui HTTP.

Injeksi LDAP

Injeksi LDAP dimungkinkan. Hal ini memungkinkan bagi penyerang untuk melewati kontrol otentikasi, serta untuk melihat dan memodifikasi data semaunya dalam direktori LDAP.

SQL Injection - MsSQL

Ini aktif scanner upaya untuk menyuntikkan MsSQL spesifik tidur perintah ke nilai parameter dan analisis respon server waktu untuk melihat apakah tidur secara efektif dijalankan pada server (yang menunjukkan sukses SQL injection attack).

Pengungkapan sumber kode - Inklusi Berkas

Menggunakan teknik penyisipan berkas lokal untuk memindai berkas yang berisi kode sumber pada server web.

Pengungkapan Kode Sumber - Git

Menggunakan teknik repositori kode sumber Git untuk memindai file yang berisi kode sumber pada server web.

Pemindai jejak.axd

Pengujian untuk melihat apakah jejak penonton (jejak.axd) tersedia. Meskipun komponen ini mudah bagi pengembang dan pemangku kepentingan lainnya dapat membocorkan sejumlah besar informasi yang analis keamanan atau individu jahat mungkin tertarik.

Pemindai jejak.axd menargetkan teknologi berbasis microsoft: IIS, windows, ASP, dan MSSQL.

Agen Pengguna Fuzzer

Pemindai aktif ini memeriksa perbedaan respons berdasarkan Agen Pengguna yang fuzzed (eg. situs ponsel, akses sebagai Mesin Pencari Crawler). Pemindai membandingkan kode status tanggapan dan kode campuran dari tanggapan tubu dengan tanggapan asli.
