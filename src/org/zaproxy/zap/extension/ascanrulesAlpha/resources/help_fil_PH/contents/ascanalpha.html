Aktibong Pag-scan ng mga Panuntunan - alpha

Kasama sa sumusunod na mga panuntunan ang mga sumusunod na alituntunin ng kalidad ng alpha na kasama sa ganitong magdagdag:

Apache Scan Rules DoS (CVE-2011-3192)

Mga pagsusuri upang makita kung ang server ay maaaring mahina sa isyu ng Apache Range Header Denial of Service, sa pamamagitan ng paghiling ng labing-isang (11) iba't ibang mga hanay ng byte. Ang labing isang hanay ay isa pang higit sa tinatanggap ng mga patched / fixed server.

Cookie Slack Detector

Sinusuri ng mga cookies upang makita kung ang ilan ba ay walang epekto sa laki ng tugon kapag tinanggal, lalung-lalo na ang mga cookies na naglalaman ng pangalang "session" o "userid"

ELMAH Scanner

Mga pagsusuri upang makita kung ang Modyul ng Pag-log ay Error at Handler (elmah.axd) ay magagamit ng HTTP Module. Kahit na ang modyul na ito ay madaling gamitin ang mga tagapag-paunlad at iba pang mga stakeholder ay maaari din itong tumagas ng isang malaking halaga ng impormasyon kung saan isang seguridad analyst o malisyosong indibidwal maaaring interesado sa.

Ang ELMAH scanner ay nagta-target ng mga teknolohiyang batay sa Microsoft: IIS, Windows, ASP, at MSSQL.

Halimbawa ng Aktibong Scanner ng File Ito ay nagpapatupad ng isang halimbawa ng aktibong pag-scan ng panuntunan na nag-load ng mga string mula sa isang file na maaaring i-edit ng user.
Para sa higit pang mga detalye tingnan: http://zaproxy.blogspot.co.uk/2014/04/hacking-zap-4-active-scan-rules.html

Halimbawa Simple Active Scanner Ito ay nagpapatupad ng isang napaka-simpleng halimbawa ng aktibong i-scan ang panuntunan.
Para sa higit pang mga detalye tingnan: http://zaproxy.blogspot.co.uk/2014/04/hacking-zap-4-active-scan-rules.html

Pagpapahiwatig ng Wika Pagpapahayag Ang software ay gumagawa ng lahat o bahagi ng pahayag na wika (EL) na pahayag sa isang Java Server Page (JSP) na gumagamit ng eksternal na naimpluwensiyang input mula sa isang upstream component, ngunit hindi ito neutralisahin o hindi tama ang neutralizes ng mga espesyal na elemento na maaaring baguhin ang inilaan na pahayag ng EL bago ito ay pinaandar. Sa ilang mga bersyon ng Spring 3.0.5 at mas maaga, nagkaroon ng kahinaan (CVE-2011-2730) kung saan ang Mga Tag ng Mga Wika ng Pagpapahayag ay masuri nang dalawang beses, na epektibong nailantad ang anumang aplikasyon sa EL injection. Gayunpaman, kahit na para sa mga susunod na bersyon, ang kahinaan na ito ay posible pa rin depende sa pagsasaayos.

GET for POST Scanner

Ang scanner na ito ay tumatagal ng application / x-www-form-urlencodedPOST na mga kahilingan, binabago ang mga parameter mula sa POST sa GET at muling ipinapadala ang kahilingan. Kung ang Tugon ng GET ay katulad ng orihinal na sagot ng POST pagkatapos ang isang alerto ay tataas. Habang ito ay hindi kinakailangan kumatawan sa isang kahinaan sa seguridad sa sarili nito ay maaaring ipahiwatig na ang iba pang mga pag-atake o kahinaan ay maaaring pinalawak o pinadali. (Tulad ng POST nakabase sa Cross-Site Scripting (XSS) ang atake ay nagbabago saGET.)

Site ng HTTP lamang Sinusuri ng aktibong scanner kung ang isang HTTP site ay nagsilbi sa ilalim ng HTTPS.

Httpoxy

Ang mga aktibong scanner na ito ay sumusuri kung ang isang site ay gumagamit ng HTTP Proxy header na tinukoy sa kahilingan.
Nagtatakda ito ng HTTP proxy na nakikinig sa lahat ng mga interface sa isang random na itinalagang libreng port. Pagkatapos ay nagpapadala ito ng isang serye ng mga kahilingan sa target server gamit ang HTTP Proxy header na nakatakda sa bawat isa sa mga magagamit IP address at ang port na ito ay nakikinig sa. Kung ang isang kahilingan ay natanggap sa bagong port, malamang na mahina ang server.
IMPORTANTENG - ang computer na tumatakbo sa ZAP ay dapat tumanggap ng mga papasok na kahilingan sa mga di-makatwirang mga port - kung pinipigilan ng isang firewall ang mga papasok na koneksyon, hindi gagana ang panuntunang ito.

HTTPS As HTTP Scanner

Tinatangka ng aktibong scanner na ma-access ang nilalaman na orihinal na na-access sa pamamagitan ng HTTPS (SSL / TLS) sa pamamagitan ng HTTP.

LDAP Injection

LDAP Iniksyon ay maaring posible. Maaaring posible para sa isang magsasalakay na laktawan ang mga kontrol ng pagpapatunay, at upang tingnan at baguhin ang di-makatwirang data sa direktoryo ng LDAP.

SQL Injection - MsSQL

Ang aktibong scanner ay sumusubok na mag-iniksyon ng mga partikular na mga utos ng pagtulog ng MsSQL sa mga halaga ng parameter at pinag-aaralan ang oras ng pagtugon ng server upang makita kung ang pagtulog ay epektibo na isinasagawa sa server (na nagpapahiwatig ng isang matagumpay na pag-atake ng SQL iniksyon).

Source Code Disclosure - File Inclusion

Gumagamit ng mga diskarte sa pagsasama ng lokal na file upang i-scan para sa mga file na naglalaman ng source code sa web server.

Source Code Disclosure - Git

Gumagamit ng Git source code repository metadata upang i-scan para sa mga file na naglalaman ng pinag-mulan ng code sa web server.

Trace.axd Scanner

Suriin upang makita kung ang Trace Viewer (trace.axd) ay magagamit. Kahit na bahagi na ito ay maginhawa para sa mga developer at ang iba pang mga stakeholder ito ay maaaring tumagas ng malaking halaga ng impormasyon na isang analyst ng seguridad o malisyosong indibidwal maaaring interesado sa.

Ang trace.axd scanner ay nagta-target ng mga teknolohiya batay sa Microsoft: IIS, Windows, ASP, at MSSQL.

User Agent Fuzzer

Ang aktibong scanner ay nagsusuri para sa mga pagkakaiba sa tugon batay sa fuzzed User Agent (eg. mobile na mga site, ma-access bilang isang Search Engine Crawler). Ang scanner na ito ay nakukumpara sa tugon ng statuscode at ang harshcode ng tugon ng katawan ay may orihinal na tugon.
