Regras de varredura ativa - alfa

As seguintes regras de varredura ativa qualidade alfa estão incluídas neste complemento:

Detector de folga/slack de Cookie

Testa os cookies para detectar se alguns não têm nenhum efeito no tamanho da resposta quando omitidos, especialmente cookies contendo o nome "sessão" ou "userid"

Scanner ativo de arquivo exemplo

Este implementa um exemplo de regra de verificação ativa que carrega strings a partir de um arquivo que o usuário pode editar.
Para mais detalhes veja: http://zaproxy.blogspot.co.uk/2014/04/hacking-zap-4-active-scan-rules.html

Exemplo simples Scanner ativo

isso implementa a regra de verificação ativa um exemplo muito simples.
Para mais detalhes veja: http://zaproxy.blogspot.co.uk/2014/04/hacking-zap-4-active-scan-rules.html

Injeção de linguagem de expressão

o software constrói todo ou parte de uma instrução de linguagem (EL) de expressão em uma página de servidor de Java (JSP) usando influenciado externamente a entrada de um componente upstream, mas ele não neutralizar ou incorretamente neutraliza elementos especiais que poderiam modificar a instrução EL pretendida antes de ser executado. Em certas versões do Spring 3.0.5 e anteriormente, houve uma vulnerabilidade (CVE-2011-2730) na qual tags de EL podiam ser avaliadas duas vezes, e isso expunha qualquer aplicação a uma EL injection. Apesar disso, mesmo para versões posteriores, esta vulnerabilidade é ainda possível, dependendo da configuração.

Integer Overflow Error

Looks for indicators of integer overflows in compiled code that causes the web server to crash. It does this by putting out multiple strings of integers designed to try and stimulate bad responses.

LDAP Injection

LDAP Injection may be possible. It may be possible for an attacker to bypass authentication controls, and to view and modify arbitrary data in the LDAP directory.

Source Code Disclosure - File Inclusion

Uses local file inclusion techniques to scan for files containing source code on the web server.

Source Code Disclosure - Git

Uses Git source code repository metadata to scan for files containing source code on the web server.

HTTPS As HTTP Scanner

This active scanner attempts to access content that was originally accessed via HTTPS (SSL/TLS) via HTTP.

User Agent Fuzzer

This active scanner checks for differences in response based on fuzzed User Agent (eg. mobile sites, access as a Search Engine Crawler). The scanner compares the response statuscode and the hashcode of the response body with the original response.
