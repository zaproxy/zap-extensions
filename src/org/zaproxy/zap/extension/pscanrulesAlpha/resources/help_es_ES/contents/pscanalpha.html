Reglas de Escaneo pasivo - Alpha

En este complemento se incluyen las siguientes reglas de escaneo pasivo de calidad alpha:

Revelación ViewState ASP.NET

Un ViewState ASP.NET fue revelado por el servidor de la aplicación o el servidor web

Integridad ViewState ASP.NET

La aplicación no usa un Código de Autenticación de Mensajes (MAC) para proteger la integridad del ViewState ASP.NET, que puede ser manipulado por un cliente malintencionado

Base64 Disclosure

Los datos codificados en Base64 fueron revelador por el servidor de la aplicación o el servidor web

Encabezado Content Security Policy faltante

Esto comprueba los encabezados de respuesta HTML para la presencia de un encabezado Content Security Policy.
Por defecto esta regla comprueba la presencia del encabezado "Content-Security-Policy" header", y también comprueba en el umbral bajo los encabezados "X-Content-Security-Policy" y "X-WebKit-CSP".

Si se encuentra un encabezado "Content-Security-Policy-Report-Only" en una respuesta, aparece una alerta INFO. Esto puede representar un esfuerzo de ejecución que esta siendo activamente refinado o desarrollado, o alguno que sea solo parcialmente implementado.

Envenenamiento de cookie

Esta comprobación observa la entrada suministrada por el usuario en los parámetros de cadena de consulta y datos POST para identificar donde se pueden controlar los parámetros de cookie. Esto se llama ataque de envenenamiento de cookie, y se vuelve vulnerable cuando un atacante puede manipular la cookie de varias maneras. En algunos casos esto no será vulnerable, sin embargo, permitir que los parámetros URL configuren valores de cookie se considera generalmente un error.

Cookie sin atributo SameSite

Esto informa cualquier cookie que no tenga el atributo SameSite y que no tenga un valor válido reconocido para ese atributo.

Cross Domain Misconfiguration

Escanea pasivamente las respuestas de Cross Domain MisConfigurations, lo cual le quita presión a Same Origin Policy en el navegador web.
La implementación actual observa los encabezados CORS excesivamente permisivos.

Navegación de Directorio

Escanea pasivamente las respuestas de firmas que indican que la navegación por el escritorio es posible.

Revelación de Hash

Escanea pasivamente hashes de contraseñas reveladas por el servidor web.
Se incluyen varios formatos, como MD4, MD5, y SHA*, que se utilizan a veces para otros propósitos distintos a contener los hashes de contraseñas.

HeartBleed

Escanea pasivamente las respuestas de encabezado HTTP que pudieran indicar que el servidor está vulnerable a la vulnerabilidad HeartBleed OpenSSL.

HTTP a transición insegura HTTPS en formulario POST

Esto comprueba las páginas HTTP inseguras que albergan formas HTTPS. El problema es que una página HTTP insegura puede ser fácilmente secuestrada a través de MITM y la forma segura HTTPS puede ser reemplazada o burlada.

HTTPS a transición insegura HTTP en formulario POST

Esto comprueba las páginas HTTPS inseguras que albergan formas HTTP inseguras. El problema es que una página segura hace la transición a una página insegura cuando los datos son cargados a través de un formulario. El usuario podría pensar que están enviando datos a una página segura cuando en realidad no es así.

Redicrección Abierta

Las redirecciones abiertas en encuentran en el Top 10 de vulnerabilidades de OWASP 2010. Esta comprobación observa la entrada generada por el usuario en cuánto a los parámetros de cadena de consulta y datos tipo POST, para identificar donde pueden ser posibles las redirecciones abiertas. Las redirecciones abiertas ocurren cuando una aplicación permite entradas suministradas por el usuario (por ejmplo: http://nottrusted.com) para controlar una dirección una redirección fuera de sitio. Esta es generalmente una manera bastante eficaz para encontrar donde 301 o 302 redirecciones podrán ser explotadas por los spammers o de los atacantes al momento de pescar

Escáner de encabezado estricto de seguridad de transporte

Éste escáner comprueba las respuestas HTTPS por la presencia de un escáner de encabezado estricto de seguridad y transporte:
  • encabezado de seguridad estricta del transporte No establece: si la respuesta es HTTPS y el encabezado se encuentra completamente extraviado.
  • Seguridad estricta de transporte desactivada: si la respuesta es HTTPS y el valor máximo para la edad se encuentra establecido en cero, restableciendo así la información del navegador HSTS para el sitio.
  • Múltiples entradas de encabezado de Seguridad y transporte estricta. (no conforme con especificaciones): si la respuesta es HTTPS y exíste mas de una cabecera HSTS en la respuesta.
  • -Seguridad de transporte estricta, falta de edad máxima (no conforme con especifiaciones): Si la respuesta es HTTPS, un encabezado HSTS es especificado pero el valor del máximo de edad no contiene un número.
  • Seguridad de transporte estricta definida por la META (no conforme con especificaciones): Si el cuerpo de respuesta incluye una etiqueta META que trata de definir una HSTS.
  • Encabezado de seguridad de transporte estricta en una respuesta HTTP simple: si la respuesta es HTTP y en encabezado HSTS se encuentra presente.
  • Seguridad de transporte estricta, máximo de edad malformado (no conforme con especificaciones): Si la respuesta es del tipo HTTPS y el encabezado HSTS se encuentra presente, pero existen citas anteriores a la directiva de máxima edad (las citas se encuentran permitidas alrededor del valor de máxima edad, pero no alrededor de la directiva como tal).
  • Seguridad de transporte estricta, contenido malformado (no conforme con las especificaciones): Si se tiene una respuesta HTTPS y un encabezado HSTS presente, pero hay algún contenido inesperado (como las citas rizadas). La expectativa es que el contenido sea imprimible según la ASCII.

Atributo de un elemento HTML controlable por el usuario (con potencial XSS)

Esta comprobación observa a la entrada generada por usuarios en cuanto a parámetros de cadena oculta y datos POST para identificar a donde ciertos atributos HTML pueden ser controlados. Esto proporciona detección de punto caliente para los XSS (cross-site scripting) que requerirá futuras revisiones por parte de analistas de seguridad en orden de determinar la explotabilidad.

Juego de caracteres controlado por el usuario

Esta comprobación observa la entrada generada por el usuario en una línea de código script y los datos tipo POST para identificar a donde el tipo de contenido o las declaraciones de forma de juego de caracteres podrían ser controlados por el usuario directamente. Estas declaraciones sobre juegos de caracteres deben ser declaradas siempre con la aplicación. Si un atacante puede controlar los juegos de caracteres, ellos podrían manioular el HTML para producir así XSS. u otros ataques.

Evento de Javascript controlado por el usuario (XSS)

Esta comprobación observa las entradas generadas por el usuario en parámetros curvos y datos POST para encontrar donde están siendo controlados. Esto proporciona detección de punto caliente para los XSS (cross-site scripting) requerirá de análisis futuros para determinar la explotabilidad.

Propiedad de Javascript controlable con el usuario (XSS)

Esta comprobación observa las entradas generadas por los usuarios en los parámetros de cadena ocultos y los datos POST para identificar donde pueden ser controladas ciertas propiedades de las URL en javascript (por ejemplo createElement src). Esto proporciona detección de punto caliente para los XXS (cross-site scripting) la cual requerirá de futuras revisiones de parte de un analista de seguridad para determinar su explotabilidad.

Hash de nombre de usuario encontrado

Si cualquier contexto contiene usuarios definidos, este escaner comprueba todas las respuestas para la presencia de valores tipo hash representados en esos nombres de usuario. El descubrimiento de cualquiera de esos valores puede representar una vulnerabilidad por una referencia insegura de objeto directo (IDOR). Las alertas son sólo planteadas como elementos informacionales, ya que se requiere de más pruebas para poder confirmar con certeza y evaluar su impacto.

Redirección de grandes

Éste clic predice el tamaño de varias respuestas de tipo redirecto y genera una alerta si la respuesta es mejor que el tamaño predicho. Una larga respuesta redireccionada grande puede indicar que se ha producido una redirección, la página actualmente contiene contenido (que puede revelar información sensible, PII, etc).

Fuga de información de la X-Depuración-Token

Esto comprueba encabezados de las respuestas de la presencia de un X-Token-depurar y de los detalles de X-Debug-Token-enlace. Lo cual indica el uso o desuso del Symfony´s Profiler. Las Symfony´s Profiler proveen acceso a una cantidad significativa de información de interes para los individuales y a los probadores de seguridad.
