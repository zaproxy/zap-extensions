Mga Patakaran sa Aktibong Pag-scan ng DOM XSS

Ang patakaran ng aktibong pag-scan para sa pagtuklas ng mga kahinaan ng DOM XSS.

Ito ay naglulunsad ng mga window ng browser at nagpapadala ng mga atake sa kargamento sa lahat ng mga nauugnay na mga elemento ng DOM.
Habang ito ay naglulunsad ng mga window ng browser, hindi ito angkop para sa headless environments, at aabot ng mas matagal kaysa sa iba (hindi batay sa browser) mga patakaran.

Ang bersyong ito ay sumusuporta lamang ng Firefox.
Ang mga bersyon sa hinaharap ay maaaring sumuporta ng ibang mga browser.

Ang mga sumusundo ng mga Kalakasan ng Atake ay suportado, at tuwirang may kinalaman sa bilang ng mga kargamento ng atake na ginamit:

  • LOW: 2 mga kargamento ng atake
  • MEDIUM: 4 mga kargamento ng atake
  • HIGHT: 6 mga kargamento ng atake
  • INSANE: 8 mga kargamento ng atake

Ang patakaran ay magtatala lamang ng isang kahinaan ng DOM XSS kada noda, maliban kung ang LOW Alert threshold ay ginamit, kung saan susubukan nito ang lahat ng tinukoy na mga kargamento.
