Rastreador de errores

The Bug Tracker add-on can help the user raise the Alerts that they receive in the form of an issue or a bug on a Bug Tracker (like Bugzilla, Github,etc.) while scanning a web-app.

Configuración de Rastreadores de Errores

The user may configure the Bug Trackers with their important details like Username, Password, Repository URL, etc. by going to Options->Bug Tracker and then selecting the appropriate Bug Tracker.

Mostrar los incidentes

El usuario puede seleccionar una o más Alertas (incluso en los subnodos de ser necesario), después de escanearlos el usuario puede elegir Elemento de menú de contexto -> Mostrar incidente en un Rastreador de Errores
Un nuevo dialogo emergerá que le dará al usuario la habilidad de editar los campos o incluso elegir algunos campos de la configuración existente. Por ejemplo: Si se elige la configuración de GIthub entonces no es necesario llenar el Nombre de Usuario, Contraseña, y URL del Repositorio siempre y cuando sean correctos. Hay un menú desplegable llamado "Elegir un Rastreador de Errores" en el cual el usuario puede elegir en cual rastreador de errores mostrar el incidente.
Los campos para mostrar incidentes dependen del Rastreador de Errores y su implementación.
Después de llenar los campos el usuario puede hacer clic en "Guardar" para mostrar el incidente. Un mensaje será mostrará al usuario indicando si el intento de mostrar el incidente fue exitoso o no, y si no lo fue, muestra la razón de ello.
