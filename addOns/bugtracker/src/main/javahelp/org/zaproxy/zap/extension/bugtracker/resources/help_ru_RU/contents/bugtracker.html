Баг трекер

Надстройка Bug Tracker может помочь пользователю поднять оповещения, которые они получают в виде проблемы или ошибки в Bug Tracker (например, Bugzilla, Github и т. д.) при сканировании веб-приложения.

Настройка баг-трекеров

The user may configure the Bug Trackers with their important details like Username, Password/Token, Repository URL, etc. перейдя в Параметры->Отслеживание ошибок, что привело к выявлению выявления ошибок.

Поднятие вопросов

Пользователь может выбрать одно или несколько предупреждений (даже их подузлы по мере необходимости) после сканирования, затем пользователь может выбрать Пункт контекстного меню -> Поднять проблему в средстве отслеживания ошибок.
Появится новое диалоговое окно, которое даст пользователю возможность редактировать поля или даже выбрать некоторые поля из существующей конфигурации. Например: если выбрана конфигурация Github, нет необходимости заполнять имя пользователя, пароль и URL-адрес репозитория, если они верны. Существует раскрывающийся список под названием «Выберите средство отслеживания ошибок», в котором пользователь может выбрать средство отслеживания ошибок, в котором нужно поднять проблему.
Поля для поднятия вопроса зависят от системы отслеживания ошибок и их реализации.
После заполнения полей пользователь может нажать «Сохранить», чтобы поднять вопрос. Затем пользователю будет показано сообщение о том, была ли его / ее попытка поднять проблему успешной или нет, и если нет, то причина этого.
