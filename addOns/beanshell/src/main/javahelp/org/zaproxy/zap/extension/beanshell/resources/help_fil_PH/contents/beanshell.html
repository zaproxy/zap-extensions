Bean Shell Console

Ang BeanShell ay isang Interactive na Java shell na maaring gamitin para paganahin ang Beanshell na mga iskrip. Ang mga Iskrip na ito ay pormang simple ng Java na gumagamit ng maraming elemento mula sa Java syntax, ngunit na sa pinaka simpleng scripting na format. All Java code is also valid BeanShell code.
BeanShell integration in OWASP ZAP enables you to write scripts using the ZAP functions and data set. Ito ay maaring maging isang napakalakas na tampok para sa pagsusuri ng mga web aplikasyon.

Ang BeanShell Console

Ang console ay nag-uumpisa mula sa Tool menu, at naglalaman ng isang split screen na kung saan ang itaas na bahagi ay ang interactive BeanShell console at ang ibabang bahagi ay isang simpleng text editor. Para sa mga complex na mga script, ikaw ay hinimuk na gamitin ang Java editor. Ang mga script ay pwedeng ma-load, naligtas at nasuri mula sa editor. Kapag ang BeanShell ay nagsimula sa isang numero ng mga bagay mula sa ZAP ay magagamit para sa paggamit, namely:

  • ang Modelo singleton, sa pamamagitan ng object na nakapangalanmodelo
  • ang SiteTree tree sa kasalukuyang mga site sa pamamagitan ng mga site na object
  • isang instance sa 
    Ang HttpSender
    sa pamamagitan ng ang nagpadala object
Pansinin na ang BeanShell ay maluwag na uri. Samakatuwid, ito ay hindi kinakailangan upang ipahayag ang mga variable bago gamitin ang mga ito - dahil ito ang mga script na medyo maikli kaysa regulat na Java. At siyempre, kung hindi mo gusto itukoy ang uri ay pwede.

Ang paggamit sa Mapa sa site

Tayo ay mag-umpisa ng anumang magagamit at tipikal: Ulitin sa pamamagitan ng lahat ng mga site node at suriin sa mga meron na sa file. Ang script ay nalikha na at natapos na ito, piliin ang Load at pumili sa example.tree.bsh na file. Ang Simpleng HTTP na hiling

Sa susunod na halimbawa, tayo ay maglikha at magpadala ng HTTP na tugon nang direkta sa interactive console: Upang lubos na magamit ang kapangyarihan ng BeanShell, ikaw ay dapat pag-aralan ang ZAP na mga panloob. Ang nagpadala na object ay parehong instance sa paggamit ng Manual Request Editor at nga awtomatiko na gumamit ng proxy na mga setting na nagtakda sa ZAP na configurasyon.

Mga tip

Gamitin ang unset(String) command para ma-unset ang anumang idineklarang mga variable, mga pamamaraan o mga bagay. Ito ay kapaki-pakinabang kung gusto mong palitan ang isang paraan na pagpapahayag sa kasalukuyang namespace. Tandaan na ang utos ay tumatagal ng isang String argumento, hindi isang bagay, kaya sa pag-unset sa bagay na ginagamit natin sa itaas, ito ay dapat: unset(“t”); at hindi unset(t);
