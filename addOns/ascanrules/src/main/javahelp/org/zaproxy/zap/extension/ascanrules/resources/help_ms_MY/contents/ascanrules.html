Active Scan Rules

The following release quality active scan rules are included in this add-on:

Buffer Overflow

Looks for indicators of buffer overflows in compiled code. Ini dilakukan dengan memadamkan rentetan teks input besar dan mencari kod ranap dan penutupan sesi yang tidak normal.

Code Injection

This rule submits PHP and ASP attack strings as values for URL parameters in a request and examines the response to see if those values have been evaluated by the server. Pertama, permintaan dibuat dan dihantar mengandungi arahan PHP yang disuntik untuk mencetak tanda. Sekiranya berlaku perlanggaran antara token dan kandungan badan tindak balas, pengimbas menimbulkan amaran dan akan segera kembali. If there aren't any matches, the scanner will construct requests with several injected ASP strings that instruct the server to write the product of two randomly generated integers in the response. Jika badan tindak balas sepadan dengan produk, amaran akan dibangkitkan.

Command Injection

This rule submits *NIX and Windows OS commands as URL parameter values to determine whether or not the web application is passing unchecked user input directly to the underlying OS. Rentetan suntikan terdiri daripada meta-aksara yang dapat ditafsirkan oleh OS sebagai perintah bergabung bersama dengan muatan yang harus menghasilkan output sebagai tindak balas jika aplikasi itu terdedah. Jika kandungan badan tindak balas sepadan dengan muatan, pengimbas menimbulkan peringatan dan mengembalikan dengan serta merta. Dengan tidak adanya usaha padanan berasaskan kesilapan yang menghasilkan output sebagai tindak balas, pengimbas akan mencuba serangan suntikan buta dengan menghantar arahan tidur sebagai tuduhan dan membandingkan masa berlalu antara permintaan penghantaran dan menerima respons kepada masa heuristik. -delay had yang lebih rendah. Sekiranya masa berlalu lebih besar dari batas ini, amaran akan meningkat dengan keyakinan sederhana dan pengimbas akan kembali.
Post 2.5.0 you can change the length of time used for the blind injection attack by changing the rules.common.sleep parameter via the Options 'Rule configuration' panel.

Client Browser Cache

This rule checks the headers of secure pages and reports an alert if they allow a browser to cache the page.

Cross Site Scripting (reflected)

This rule starts by submitting a 'safe' value and analyzing all of the locations in which this value occurs in the response (if any).
It then performs a series of attacks specifically targeted at the location in which each of the instances occurs, including tag attributes, URL attributes, attributes in tags which support src attributes, html comments etc.

Cross Site Scripting (persistent)

This rule starts by submitting a unique 'safe' value and then spiders the whole application to find all of the locations in which the value occurs.
It then performs a series of attacks in the same way that the 'reflected' version does but in this case checks all of the target locations in other pages.

CRLF Injection

This rule submits various CRLF special characters preceding an injected "Set-Cookie" header as a parameter to the server. Jika respons mengandungi header Set-Cookie yang sama, amaran akan dibangkitkan dan pengimbas akan kembali dengan serta merta.

Directory Browsing

This rule checks to see if a request will provide access to a directory listing by examining the response body for patterns used with Apache, IIS and other web server software.

External Redirect

This rule submits a variety of URL redirect strings as parameter values in a request, then examines the headers and bodies of responses to determine whether or not a redirect occurred and of what type. Penyebab redirect dicari dalam lajur header "Lokasi" dan "Menyegarkan", serta tag meta HTML dan Javascript dalam badan respons. Amaran yang dibangkitkan termasuk jenis pengalihan dan pengimbas akan kembali.

Format String Error

Looks for indicators of format string handling errors in compiled code. Ini dilakukan dengan meletakkan rentetan teks masukan berasaskan karakter yang disusun kod C yang menjangkakan untuk menghasilkan output yang diformat dan mencari kod ranap dan penutupan sesi yang tidak normal.

Parameter Tampering

This rule submits requests with parameter values known to cause errors to be displayed to the user if handled improperly. Respon diperiksa untuk memastikan bahawa mereka mengembalikan kod ralat status pelayan, kemudian dibandingkan dengan respons HTTP biasa untuk memastikan ia tidak menyebabkan amaran jika parameter buruk tidak mempunyai kesan pada output. Akhirnya, kandungan tindak balas badan dibandingkan dengan pelbagai corak yang terdapat dalam servlet Java, kesilapan Microsoft VBScript, OLE DB, JET, PHP dan Tomcat. Sekiranya terdapat perlawanan, amaran akan dinaikkan dan pengimbas akan kembali tidak lama lagi.

Path Traversal

This rule attempts to access files and directories outside of the web document root by constructing various combinations of pathname prefixes and local file targets for Windows and *NIX systems as well as Java servlets. Jika badan bertindak balas mengikut corak yang sepadan dengan fail sasaran semasa, amaran akan dibangkitkan dan pengimbas akan dikembalikan tidak lama lagi. Sekiranya tiada sasaran fail setempat yang biasa berjaya, jalan traversal dicuba menggunakan nama fail dalam URL. Selagi menghantar nama fail sewenang tidak mengembalikan kod status OK tetapi nama fail sebenarnya tidak, amaran akan dinaikkan dan pengimbas akan kembali tidak lama lagi.

Remote File Include

This rule submits a series of requests with external URLs as parameter values and looks for a match between the the response body and the title of the page hosted at those URLs. Sekiranya terdapat padanan di antara rentetan yang dijangkakan dan badan itu dijawab, dan tajuk itu mengembalikan kod status 200, amaran akan diangkat dan pengimbas tidak lama lagi akan kembali.

Server Side Include

This rule checks to see what OS the server is running on, then sends requests with a corresponding HTML SSI directive as a parameter value. Jika badan bertindak balas mengikut corak yang menunjukkan SSI berjaya, amaran akan dinaikkan dan pengimbas akan kembali dengan segera.

SQL Injection

This scanner scans for SQL Injection vulnerabilities in an RDBMS-independent fashion, by attacking url parameters and form parameters with fragments of valid and invalid SQL syntax, using error based, boolean based, Union based, and stacked query SQL Injection techniques.
This scanner may be able to fingerprint the RDBMS if the application throws a known RDBMS specific SQL error message.
This scanner does not exploit any RDBMS specific techniques, and so is the best SQL injection scanner to use as a starting point.
