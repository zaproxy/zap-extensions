Aktifkan Aturan Pemindaian

Berikut kualitas perilisan aktif aturan-aturan pemindaian termasuk tambahan ini:

Luapan Penyangga

untuk melihat indikator-indikator dari luapan dalam kode terkompilasi. Hal ini dilakukan dengan menempatkan string teks masukan yang besar dan mencari kode yang rusak dan penutupan sesi abnormal.

Kode injeksi

Aturan ini diajukan PHP dan ASP menyerang senar sebagai nilai untuk URL parameter dalam sebuah permintaan dan memeriksa respon untuk lihat jika nilai itu telah dievaluasi oleh server. Pertama, permintaan dibuat dan dikirim berisi petunjuk PHP yang disuntikkan untuk mencetak token. Jika terjadi benturan antara token dan isi dari respons body, pemindai menimbulkan peringatan dan segera kembali. Jika tidak ada pertandingan, pemindaian akan membuat permintaan dengan beberapa string ASP yang disuntikkan yang menginstruksikan server untuk menulis produk dari dua bilangan bulat yang dihasilkan secara acak dalam responnya. Jika respon tubuh cocok dengan produk, peringatan akan dinaikan.

Injeksi Perintah

aturan ini mengajukan *NIX dan Windows OS perintah sebagai nilai parameter URL untuk menentukan apakah atau tidakkah aplikasi web dilewati tidak di periksa input pengguna secara langsung untuk OS yang mendasarinya. String injeksi terdiri dari meta-karakter yang bisa diinterpretasikan oleh OS sebagai perintah join beserta payload yang seharusnya menghasilkan output dalam respon jika aplikasi tersebut rentan. Jika isi dari response body sesuai dengan muatan, pemindai menimbulkan peringatan dan segeralah kembali. Dalam hal ini tidak ada kesalahan berdasar yang sesuai berusaha mengembalikan output dalam respon scanner akan melakukan serangan injeksi buta dengan mengajukan instruksi tidur sebagai muatan dan membandingkan waktu yang berlalu antara mengirimkan permintaan dan menerima respon terhadap batas bawah waktu tunda heuristik. Jika waktu berlalu lebih cepat dari batas ini, peringatan akan meningkat dengan kepercayaan diri sedang dan pemindai segera kembali.
Post 2.5.0 anda dapat mengubah jangka waktu yang digunakan untuk serangan injeksi buta dengan mengganti parameter rules.common.sleep melalui Opsi panel 'Rule configuration'.

Cache Browser Klien

Aturan ini memeriksa header dari keamanan halaman dan laporan-laporan sebuah peringatan jika mereka mengizinkan sebuah browser untuk halaman cache.

Cross Site Scripting (refleksi)

Aturan ini dimulai dengan mengirimkan sebuah nilai 'safe' dan menganalisis semua dari lokasi dimana nilai terjadi di dalam respon (jika ada).
Ini kemudian melakukan seri dari serangan khususnya ditargetkan di lokasi di mana masing-masing kasus terjadi, termasuk menandai atribut, URL atribut, atribut dalam menandai yang mendukung src atribut, html komentar etc.

Penaskahan Situs Sebrang (tetap)

Aturan ini dimulai oleh sebuah nilai unik yang 'aman' dan kemudian laba-laba keseluruhan aplikasi untuk menemukan semua lokasi di mana nilainya terjadi.
Kemudian melakukan serangkaian serangan dengan cara yang sama seperti versi 'tercermin' namun dalam kasus ini memeriksa semua target lokasi dalam halaman lain.

Injeksi CRLF

Aturan ini mengajukan variasi-variasi karakter spesial CRLF sebelum header "Set-Cookie" yang disuntikkan sebagai parameter ke server. Jika tanggapan berisi header Set-Cookie yang identik, peringatan akan dinaikkan dan pemindai akan segera kembali.

Direktori Browsing

Aturan ini memeriksa untuk melihat jika permintaan akan menyediakan akses ke daftar direktori dengan memeriksa respon badan untuk pola yang digunakan dengan Apache, IIS dan software server web lain.

Pengalihan Eksternal

Aturan ini mengajukan sebuah variasi dari URL pengalihan syarat sebagai nilai parameter dalam sebuah permohinan, kemudian gabungkan antara kepala dan tubuh dari respon-respon untuk menentukan apakah atau tidakkah sebuah pengalihan terjadi dan dari jenis apa. Penyebab redirect adalah mencari di "Lokasi" dan "Refresh" header kolom, serta dengan HTML meta tag dan Javascript dalam tubuh respon. Peringatan dimunculkan termasuk jenis pengalihan dan pemindai mengembalikan segera.

Kesalahan Format string

mencari indikator dari format kesalahan penanganan string dalam kode dikompilasi. Hal ini dilakukan dengan meletakkan string input teks berdasarkan karakter yang dikompilasi kode C yang mengantisipasi untuk output produk yang diformat dan mencari kode rusak dan penutup sesi yang tidak normal.

Gangguan Parameter

Aturan ini mengajukan permohonan dengan nilai parameter untuk mengetahui sebab kesalahan untuk ditampilkan untuk pengguna jika pengendalian tidak layak. Respon diperiksa untuk memastikan bahwa mereka mengembalikan Kode status kesalahan server, kemudian membandingkan dengan respon HTTP normal untuk memastikannya tidak memunculkan peringatan parameter buruk tidak berpengaruh pada output. Akhirnya, konten respon tubuh dibandingkan dengan berbagai pola yang mungkin ditemukan di Java servlet, Microsoft VBScript, OLE, JET, PHP dan kesalahan Tomcat. Jika kecocokan ditemukan, peringatan akan muncul dan pemindai mengembalikan segera.

Jalan Traversal

Aturan ini mencoba mengakses file dan direktori diluar dari akar web dokumen oleh berbagai bangunan kombinasi oleh jalan nama awalan dan target file lokal untuk windows dan *NIX sistem sama seperti Java servlets. Jika respon tubuh sesuai dengan pola yang sesuai dengan berkas target saat ini peringatan dimunculkan dan pemindai mengembalikan segera. Jika tidak ada umum setempat berkas target berhasil, jalan traversal adalah percobaan menggunakan nama berkas di URL. Selama mengirimkan nama file sewenang-wenang tidak mengembalikan kode status OK tapi nama file yang benar melakukannya, peringatan dimunculkan dan pemindai mengembalikan segera.

Termasuk File Terpencil

Aturan ini mengajukan sebuat seri dari permohonan dengan URLs eksternal sebagai nilai parameter dan mencari sebuah kecocokan host pada URLs itu. Jika ada kecocokan antara string yang diharapkan dan respon badan, dan header mengembalikan kode status 200, peringatan dimunculkan dan pemindai Mengembalikan segera.

Termasuk Server Samping

Aturan ini memeriksa untuk melihat apa server OS yang sedang di jalankan, kemudian mengirim permohonan dengan dengan direktif HTML SSI sesuai sebagau nilai parameter. Jika respon badan sesuai dengan pola yang menunjukkan SISI berhasil, peringatan dimunculkan dan pemindai mengembalikan segera.

Injeksi SQL

Pemindai ini memindai kerentanan SQL Injection dengan cara RDBMS-independen, dengan menyerang parameter url dan formulir parameter dengan fragmen sintaks SQL yang valid dan tidak valid, menggunakan kesalahan dasar, dasar boolean, kesatuan dasar, dan di tumpuk teknik-teknik injeksi SQL.
Pemindai ini mungkin bisa untuk sidik jari RDBMS jika aplikasi melemparkan yang diketahui RDBMS tertentu kesalahan pesan.
Pemindai ini tidak memanfaatkan apapun RDBMS cara tertentu, dan begitu juga yang terbaik SQL suntikan pemindai untuk digunakan sebagai titik awal.
