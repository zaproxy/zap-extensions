Akses Kontrol Konteks Pilihan

Akses Kontrol Konteks pilihan hadir sebagai panel untuk masing-masing Konteks ketika membuka Sesi dialog Properties. Panel ini memungkinkan pengguna ZAP untuk menentukan Aturan Akses untuk setiap Pengguna masing-masing Konteks.

Seperti yang disebutkan pada halaman konsep, ZAP sedang menyusun penggunaan dari pohon-berdasarkan struktur dari URL. Jadi, ketika mengkonfigurasi aturan akses, hanya 1 aturan yang harus ditetapkan secara eksplisit untuk seluruh sub pohon, sedangkan untuk node lain aturannya disimpulkan. Tiga nilai yang mungkin dapat diatur untuk setiap node dalam Konteks untuk setiap Pengguna:

  • Diizinkan - Peraturan untuk node itu secara eksplisit diatur sebagai Diizinkan
  • Ditolak - Peraturan untuk node itusecara eksplisit diatur sebagai Ditolak
  • Warisan - Aturan untuk simpul disimpulkan berdasarkan induk terdekat dengan aturan yang ditetapkan secara eksplisit.

Secara default, aturan untuk setiap node memiliki nilai Warisan dan simpul akar memiliki nilai Tidak diketahui . Dengan demikian, untuk setiap node tanpa orang tua dengan aturan yang secara eksplisit dikonfigurasi, aturan yang disimpulkan akan menjadi default Tidak diketahui.
