Options du Contexte de Contrôle d’Accès

Les Options de contexte du Contrôle d’Accès sont disponibles sous forme de panneau pour chaque contexte lors de l’ouverture de la boîte de dialogue Propriétés de session. Ce panneau permet aux utilisateurs de ZAP de définir les règles d’accès pour chaque utilisateur de chaque contexte.

Comme mentionné sur la page concepts, ZAP utilise une structure d’URL en arborescence. Ainsi, lors de la configuration des règles d’accès, une seule règle doit être définie explicitement pour un sous-arbre entier, tandis que pour les autres nœuds, les règles sont déduites. Trois valeurs possibles peuvent être définies pour n’importe quel nœud du contexte, pour chaque utilisateur

  • Allowed - The rule for the node is explicitly set as Allowed
  • Denied - The rule for the node is explicitly set as Denied
  • Inherited - The rule for the node is inferred based on the closest parent with an explicitly set rule

By default, the rules for any node has an Inherited value and the root node has a value of Unknown. Thus, for any node without a parent with an explicitly configured rule, the inferred rule will default to Unknown.
