Платформа автоматизации — заданиеpassiveScan-config

Это задание позволяет управлять конфигурацией пассивного сканирования.

Пассивный сканер работает со всеми запросами и ответами, которые генерируются ZAP или передаются через него. Если вы хотите настроить конфигурацию пассивного сканирования, обычно это следует делать перед запуском любых других заданий. Однако вы можете запустить это задание позже или несколько раз, если хотите, чтобы разные задания использовали разные конфигурации пассивного сканирования.

YAML

  - type: passiveScan-config           # Конфигурация пассивного сканирования 
    parameters:
      maxAlertsPerRule: 10             # Int: Максимальное количество оповещений для каждого правила 
      scanOnlyInScope: true            # Bool: Сканировать только URL-адреса в области действия (рекомендуется) 
      maxBodySizeInBytesToScan:        # Int: Максимальный размер тела для сканирования, по умолчанию: 0 — будут сканироваться все сообщения 
      enableTags: false                # Bool: Включить теги пассивного сканирования, по умолчанию: false — их включение может повлиять на производительность. 
    rules:                             # Список одного или нескольких правил пассивного сканирования и связанных с ними параметров, которые переопределяют значения по умолчанию. 
    - id:                              # Int: Идентификатор правила согласно https://www.zaproxy.org/docs/alerts/ 
      name:                            # String: Имя правила для целей документирования — оно не требуется или фактически не используется. 
      threshold:                       # String: Порог оповещения для этого правила: Выкл, Низкий, Средний, Высокий, по умолчанию: Средний
