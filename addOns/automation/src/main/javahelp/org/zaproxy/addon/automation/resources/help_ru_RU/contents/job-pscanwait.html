Платформа автоматизации — задание пассивного сканирования с ожиданием

Это задание ожидает, пока пассивный сканер завершит сканирование запросов и ответов в текущей очереди. Обычно это задание следует запускать после заданий, которые исследуют ваше приложение, таких как задания паука или те, которые импортируют определения API. Если какие-либо другие запросы отправляются ZAP или проксируются через ZAP после выполнения этого задания, они будут обработаны пассивным сканером. Вы можете запускать эту работу столько раз, сколько вам нужно.

YAML

  - type: passiveScan-wait             # Пассивный сканер дождитесь завершения пассивного сканера 
    parameters:
      maxDuration: 5                   # Int: Максимальное время ожидания пассивного сканера, по умолчанию: 0 неограниченно

Данные задания

Следующий класс будет доступен для надстроек, предоставляющих доступ к данным задания, таких как надстройка Reporting. Обратите внимание, что в этом случае данные взяты из включенных в данный момент правил пассивного сканирования, независимо от того, использовались ли они в результате Automation Framework, пользовательского интерфейса или API.
  • Ключ: passiveScanData
  • Класс: PassiveScanJobResultData
