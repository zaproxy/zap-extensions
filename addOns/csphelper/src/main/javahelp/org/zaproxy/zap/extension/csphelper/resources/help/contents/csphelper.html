CSP Helper

This add-on generates a Content Security Policy (CSP) Header for sites and allows you to test it without making any changes to the site.

In order to use it:

  1. Add the site you want to generate CSP for to ZAP, for example by proxying your browser through ZAP or by using one of the ZAP Spiders
  2. Locate the site in the Sites tree, right click it and select 'Enable for CSP Helper'
  3. Explore the site as thoroughly as possible
When you enable the CSP Helper for a site ZAP will generate and inject a CSP header into the HTML returned to your browser.
It will update the CSP header to permit the minimum set of functionality that will let your site continue to function correctly.
It also injects a CSP violation report URL which will cause any CSP violations to be sent to ZAP. If you use this add-on with a release after 2.5.0 (which includes the weekly releases) then ZAP will be able to analyse these reports. A restriction in ZAP releases up to and including 2.5.0 prevents this from working. It updates the CSP header based on:
  • Parsing each of the requests from the site
  • Analysing requests to other sites that include a Referer header from the target site
  • Analysing any CSP reports generated (post 2.5.0)
