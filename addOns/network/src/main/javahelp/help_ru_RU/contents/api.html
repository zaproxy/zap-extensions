Сетевой API

В API добавлены следующие операции:

Действия

  • generateRootCaCert: создает новый корневой сертификат ЦС ( Root CA) , используемый для выдачи сертификатов сервера.
  • importRootCaCert (filePath*): импортирует корневой сертификат ЦС ( Root CA ), который будет использоваться для выдачи сертификатов сервера.
    • filePath: путь в файловой системе к файлу PEM, содержащему сертификат и закрытый ключ.
  • Примечание. Эта конечная точка доступна только в еженедельных выпусках и версиях после 2.11.
    setRootCaCertValidity (validity*): устанавливает срок действия сертификата корневого ЦС ( Root CA ). Используется при создании нового сертификата Root CA.
    • срок действия: количество дней, в течение которых будет действителен сгенерированный корневой сертификат CA (Root CA).
  • Примечание. Эта конечная точка доступна только в еженедельных выпусках и версиях после 2.11.
    setServerCertValidity (validity*): устанавливает действительность сертификата сервера. Используется при создании сертификатов сервера.
    • срок действия: количество дней, в течение которых сгенерированные сертификаты сервера будут действительны.

Просмотры

Примечание. Эти конечные точки доступны только в еженедельных выпусках и версиях после 2.11.
  • getRootCaCertValidity: получает срок действия сертификата корневого ЦС (Root CA certificate) в днях. Используется при создании нового сертификата Root CA.
  • getServerCertValidity: получает срок действия сертификата сервера в днях. Используется при создании сертификатов сервера.

Другой

  • rootCaCert: получает корневой сертификат ЦС ( Root CA certificate), используемый для выдачи сертификатов сервера. Подходит для импорта в клиентские приложения (например, браузеры).
