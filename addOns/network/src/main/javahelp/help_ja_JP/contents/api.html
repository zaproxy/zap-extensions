ネットワーク API

APIに以下の操作が追加されます：

アクション

  • addAlias (name* enabled): ローカルサーバー/プロキシのエイリアスを追加します。
    • name: エイリアスの名前
    • enabled: 有効状態 (true または false)
  • addHttpProxyExclusion (host* enabled): HTTPプロキシから除外するホストを追加します。
    • host: ホストの値、正規表現
    • enabled: 有効状態 (true または false)
  • addLocalServer (address* port* api proxy behindNat decodeResponse removeAcceptEncoding): ローカルサーバー/プロキシを追加します。
    • address: ローカルサーバー/プロキシのアドレス
    • port: ローカルサーバー/プロキシのポート
    • api: ZAP APIが利用可能かどうか、true または false
    • proxy: ローカルサーバーがプロキシとして動作するかどうか、true または false
    • behindNat: ローカルサーバーがNATの背後にあるかどうか、true または false
    • decodeResponse: レスポンスをデコードするかどうか、true または false
    • removeAcceptEncoding: リクエストヘッダーの Accept-Encoding を削除するかどうか、true または false。
  • addPassThrough (authority* enabled): ローカルプロキシを通過させる権限を追加します。
    • authority: 権限の値、正規表現が使用可能
    • enabled: 有効状態 (true または false)
  • addPkcs12ClientCertificate (filePath* password* index): PKCS#12ファイルに含まれるクライアント証明書を追加します。証明書は自動的にアクティブに設定され使用されます。
    • filePath: ファイルパス
    • password: ファイルのパスワード
    • index: ファイル内の証明書のインデックス、デフォルトは0
  • addRateLimitRule (description* enabled* matchRegex matchString requestsPerSecond* groupBy*): レート制限ルールを追加します。
    • description: ルールを識別するための説明。 各ルールには一意の説明が必要です。
    • enabled: 有効状態 (true または false)
    • matchRegex: matchString が正規表現かプレーンな文字列かを決定します: trueまたはfalse。
    • matchString: プレーンな文字列マッチはDNS規約に基づいて処理されます。 文字列が1つまたは2つのコンポーネントを持つ場合。 matchRegex が true の場合、これは正規表現です。
    • requestsPerSecond: 1秒あたりの最大リクエスト数
    • groupBy: レート制限を適用する際のホストのグループ化方法: rule または host
  • generateRootCaCert: サーバー証明書の発行に使用する新しいルートCA証明書を生成します。
  • importRootCaCert (filePath*): サーバー証明書の発行に使用するルートCA証明書をインポートします。
    • filePath: 証明書と秘密鍵を含むPEMファイルへのファイルシステムパス
  • removeAlias (name*): エイリアスを削除します。
    • name: エイリアスの名前
  • removeHttpProxyExclusion (host*): HTTPプロキシ除外を削除します。
    • host: ホストの値
  • removeLocalServer (address* port*): ローカルサーバー/プロキシを削除します。
    • address: ローカルサーバー/プロキシのアドレス
    • port: ローカルサーバー/プロキシのポート
  • removePassThrough (authority*): パススルーを削除します。
    • authority: 権限の値
  • removeRateLimitRule (description*): レート制限ルールを削除します。
    • description: ルールの説明
  • setAliasEnabled (name* enabled): エイリアスが有効かどうかを設定します。
    • name: エイリアスの名前
    • enabled: 有効状態 (true または false)
  • setConnectionTimeout (timeout*): 読み取りと接続のタイムアウトを設定します。
    • timeout: タイムアウト (秒)
  • setDefaultUserAgent (userAgent*): デフォルトのユーザーエージェントを設定します。
    • userAgent: デフォルトのユーザーエージェント
  • setDnsTtlSuccessfulQueries (ttl*): 成功したDNSクエリのTTLを設定します。
    • ttl: TTL (秒) 負の数の場合は永続的にキャッシュします。 0の場合はキャッシュを無効にします。 正の数の場合は成功したDNSクエリがキャッシュされる秒数。
  • setHttpProxy (host* port* realm username password): HTTPプロキシ設定を構成します。
    • host: ホスト、名前またはアドレス
    • port: ポート
    • realm: 認証レルム
    • username: ユーザー名
    • password: パスワード
  • setHttpProxyAuthEnabled (enabled*): HTTPプロキシ認証が有効かどうかを設定します。
    • enabled: 有効状態 (true または false)
  • setHttpProxyEnabled (enabled*): HTTPプロキシが有効かどうかを設定します。
    • enabled: 有効状態 (true または false)
  • setHttpProxyExclusionEnabled (host* enabled*): HTTPプロキシ除外が有効かどうかを設定します。
    • host: ホストの値
    • enabled: 有効状態 (true または false)
  • setPassThroughEnabled (authority* enabled): パススルーが有効かどうかを設定します。
    • authority: 権限の値
    • enabled: 有効状態 (true または false)
  • setRateLimitRuleEnabled (description*, enabled*): レート制限ルールの有効状態を設定します。
    • description: ルールの説明
    • enabled: 有効状態 (true または false)
  • setRootCaCertValidity (validity*): ルートCA証明書の有効期間を設定します。 新しいルートCA証明書を生成する際に使用されます。
    • validity: 生成されるルートCA証明書が有効となる日数
  • setServerCertValidity (validity*): サーバー証明書の有効期間を設定します。 サーバー証明書を生成する際に使用されます。
    • validity: 生成されるサーバー証明書が有効となる日数
  • setSocksProxy (host* port* version useDns username password): SOCKSプロキシ設定を構成します。
    • host: ホスト、名前またはアドレス
    • port: ポート
    • version: SOCKSバージョン
    • useDns: 名前解決をSOCKSプロキシで行うかどうか、trueまたはfalse
    • username: ユーザー名
    • password: パスワード
  • setSocksProxyEnabled (enabled*): SOCKSプロキシが有効かどうかを設定します。
    • enabled: 有効状態 (true または false)
  • setUseClientCertificate (use*): アクティブなクライアント証明書を使用するかどうかを設定します。
    • use: 使用状態、trueまたはfalse
  • setUseGlobalHttpState (use*): グローバルHTTP状態を使用するかどうかを設定します。
    • use: 使用状態、trueまたはfalse

ビュー

  • getAliases: ローカルサーバー/プロキシを識別するために使用されるエイリアスを取得します。
  • getConnectionTimeout: 接続タイムアウトを秒単位で取得します。
  • getDefaultUserAgent: デフォルトのユーザーエージェントを取得します。
  • getDnsTtlSuccessfulQueries: 成功したDNSクエリのTTL (秒単位) を取得します。
  • getHttpProxy: HTTPプロキシを取得します。
  • getHttpProxyExclusions: HTTPプロキシ除外を取得します。
  • getLocalServers: ローカルサーバー/プロキシを取得します。
  • getPassThroughs: ローカルプロキシを通過する権限を取得します。
  • getRateLimitRules: レート制限ルールを取得します。
  • getRootCaCertValidity: ルートCA証明書の有効期間を日数で取得します。 新しいルートCA証明書を生成する際に使用されます。
  • getServerCertValidity: サーバー証明書の有効期間を日数で取得します。 サーバー証明書を生成する際に使用されます。
  • getSocksProxy: SOCKSプロキシを取得します。
  • isHttpProxyAuthEnabled: HTTPプロキシ認証が有効かどうかを返します。
  • isHttpProxyEnabled: HTTPプロキシが有効かどうかを返します。
  • isSocksProxyEnabled: SOCKSプロキシが有効かどうかを返します。
  • isUseGlobalHttpState: グローバルHTTP状態を使用するかどうかを返します。

その他

  • proxy.pac: メインプロキシを経由してプロキシするPACファイルを提供します。
  • rootCaCert: サーバー証明書の発行に使用されるルートCA証明書を取得します。 クライアントアプリケーション(ブラウザなど)へのインポートに適しています。
  • setProxy (proxy*): HTTPプロキシ設定を構成します。
    • proxy: HTTPプロキシ設定を含むJSONオブジェクト。

ショートカット

  • proxy.pac: メインプロキシを経由してプロキシするPACファイルを提供します。
  • setproxy: HTTPプロキシ設定を構成します。
    • リクエストボディ: HTTPプロキシ設定を含むJSONオブジェクト。
