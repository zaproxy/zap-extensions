WSTG Mapper

WSTG Mapper provides a compliance dashboard that automatically maps ZAP alerts to the OWASP Web Security Testing Guide (WSTG) coverage, giving testers a persistent, session-backed view of their testing progress.

Status Panel

The WSTG Mapper panel appears in the ZAP status bar. It displays all WSTG test cases grouped by category, showing which tests have been triggered by ZAP alerts and allowing testers to manually track their status.

Coverage Tracking

ZAP alerts are automatically matched against the bundled alert-to-WSTG mapping. When an alert is raised, the corresponding WSTG test is marked as triggered. The coverage summary at the top of the panel shows the percentage of categories fully covered.

Test Statuses

Each WSTG test can be set to one of the following statuses:

  • Not Tested - Default state; the test has not been assessed.
  • Passed - The test was assessed and the application passed.
  • Failed - The test was assessed and a vulnerability was found.
  • Manual Review - The test requires manual review and cannot be automated.
  • Not Applicable - The test does not apply to the target application.

Notes

Each test has a free-text notes field in the detail panel. Notes are saved automatically when you move to another test and are persisted with the ZAP session.

Filtering and Search

The panel toolbar provides filters by status, trigger state, and technology stack, as well as a free-text search across test IDs and names.

Export

Coverage reports can be exported as Markdown or CSV from the toolbar. Individual categories can also be exported via the right-click context menu.

Session Persistence

All statuses and notes are saved in a sidecar file alongside the ZAP session file (.session.wstgmapper.xml). When a saved session is opened, the checklist state is automatically restored.
