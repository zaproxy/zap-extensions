エンコード / デコード/ ハッシュ ダイアログ

これはテキストをエンコード、デコード、またはハッシュすることを可能にします。

共通項目

このダイアログには、すべてのタブに共通の項目が1つあります：

エンコード/デコード/ハッシュするテキスト:

この項目は、エンコード、デコード、またはハッシュしたいテキストのためのものです。
ダイアログ起動時にテキストが選択されていた場合、そのテキストがこの項目に設定されます。 テキストを変更すると、他の項目が動的に更新されます

ツールバー

  • 新しいタブの追加 - ダイアログに新しいタブを追加します。
  • 選択したタブを削除 - ダイアログから選択されている (または現在の) タブを削除します。
  • 現在のタブに出力パネルを追加 - 現在のタブに出力パネルを追加します。
  • リセット - すべてのタブ/パネルを既定の状態にリセットします。
  • オプション - オプションダイアログを開き、エンコード/デコード/ハッシュオプションパネルにフォーカスを設定します。
  • ヘルプ - ユーザーガイドを開き、コンテンツペインがエンコード/デコード/ハッシュアドオンのヘルプページにプリセットされます。

出力パネルコンテキストメニュー

  • 検索 - 標準の入力ダイアログを表示し、テキストエリア内での検索を可能にします。
  • コピー - 標準のコピー機能。
  • 貼り付け - 標準の貼り付け機能。
  • 出力パネルを削除 - 関連する出力テキストエリアを削除します。
  • 入力テキストを置換 - 関連する出力テキストエリアの選択または全内容に基づいて、入力テキストの内容を置換します。 （テキストエリアに内容がある場合にのみアクティブになります。）

既定のダイアログタブとパネル

既定の状態では、エンコード/デコード/ハッシュダイアログは、次の設定でタブと出力パネルを表示します：

エンコードタブの項目

  • Base64 エンコード
  • Base64 URL エンコード
  • URL エンコード
  • 完全な URL エンコード
  • ASCII Hex エンコード
  • HTML エンコード
  • 完全な HTML エンコード
  • JavaScript エンコード

デコードタブの項目

  • Base64 デコード
  • Base64 URL デコード
  • URL デコード
  • 完全な URL デコード
  • ASCII Hex デコード
  • HTML デコード
  • JavaScript デコード

ハッシュタブの項目

  • SHA256 ハッシュ
  • SHA1 ハッシュ
  • MD5 ハッシュ

不正なUTF-8

  • 2 バイト
  • 3 バイト
  • 4 バイト

Unicode

  • エスケープされたテキスト
  • エスケープされていないテキスト

定義済みエンコード/デコード/ハッシュプロセッサ

エンコーダー

2バイトの不正なUTF-8

2バイトの不正なUTF-8文字シーケンスを生成し表示します。

3バイトの不正なUTF-8

3バイトの不正なUTF-8文字シーケンスを生成し表示します。

4バイトの不正なUTF-8

4バイトの不正なUTF-8文字シーケンスを生成し表示します。

ASCII Hex エンコード

入力したテキストのASCII 16進数エンコーディングを表示します。

Base64 エンコード

入力したテキストのBase64エンコーディングを表示します。

Base64 URL エンコード

入力したテキストのBase 64 URLエンコーディングを表示します。 Base64URL は、エンコード結果をファイル名または URL アドレスとして使用できるようにすることを目的とした、主要な Base64 標準の修正です。 Base64URLは RFC 4648 で説明されており、標準のBase64 アルファベットにはURLやファイル名として無効な文字 (例: プラス記号や疑問符が問題となる) が含まれていることが指摘されています。 メインの標準では <<???>> を PDw/Pz8+Pg== にエンコードしますが、Base64URLはこれを PDw_Pz8-Pg に変換します。

エスケープされた Unicode テキスト

エスケープされたUnicode文字を表示します。 例えば、テキスト Açores は %u0041%u00e7%u006f%u0072%u0065%u0073 とエンコードされます。

HTML エンコード

入力したテキストのHTMLエンコーディングを表示します。 例えば、テキスト "piñata" は &quot;pi&ntilde;ata&quot; とエンコードされます。

完全な HTML エンコード

入力したテキストの完全なHTMLエンコーディングを表示します。 例えば、テキスト <script> は &#60;&#115;&#99;&#114;&#105;&#112;&#116;&#62; とエンコードされます。

JavaScript エンコード

入力したテキストのエスケープされたJavaScriptリテラルを表示します。 例えば、テキスト "2￠" は \"2\uFFE0\" とエンコードされます。

URL エンコード

入力したテキストのURLエンコーディングを表示します。

完全な URL エンコード

入力したテキストの完全なURLエンコーディングを表示します (すべての文字を16進数に変換し、パーセントを前置します)。

エスケープされていない Unicode テキスト

エスケープされていないUnicode文字を表示します。 例えば、テキスト %u0041%u00e7%u006f%u0072%u0065%u0073 は Açores とデコードされます。

モールス符号エンコーダー

提供された英数字 (スペースを含む) 入力に対応する短点 (.) と長点 (-)、単語区切り (/) を表示します。 例えば、テキスト SOS SOS は ... --- .../... --- ... とエンコードされます。

デコーダー

ASCII Hex デコード

入力したテキストのASCII 16進数デコーディングを表示します。
有効なデコードがない場合は、この項目は無効になります。

Base64 デコード

入力したテキストのBase64 デコーディングを表示します。
折り返された行を処理するために、Mime デコーダーを活用します。

Base64 URL デコード

入力したテキストのBase64 URLデコーディングを表示します。 Base64URLは、エンコード結果をファイル名またはURL アドレスとして使用できるようにすることを目的とした、主要な Base64 標準の修正です。 Base64URLは RFC 4648 で説明されており、標準のBase64 アルファベットにはURLやファイル名として無効な文字 (例: プラス記号や疑問符が問題となる) が含まれていることが指摘されています。

HTML デコード

入力したテキストの HTML デコーディングを表示します。 例えば、テキスト &quot;pi&ntilde;ata&quot; は "piñata" とデコードされます。

JavaScript デコード

入力したテキストのアンエスケープされたJavaScriptリテラルを表示します。 例えば、テキスト \"2\uFFE0\" は "2￠" とデコードされます。

URL デコード

入力したテキストのURLデコーディングを表示します。

完全な URL デコード

入力したテキストのURLデコーディングを表示します (パーセント記号が削除され、16進数デコードされます)。

モールス符号デコーダー

提供されたモールス符号入力に対応する英数字 (スペースを含む) 出力で表示します。 例えば、テキスト ... --- .../... --- ... は SOS SOS とデコードされます。

ハッシュ処理ツール

MD5 ハッシュ

入力したテキストのMD5ハッシュを表示します。

SHA1 ハッシュ

入力したテキストのSHA1ハッシュを表示します。

SHA256 ハッシュ

入力したテキストのSHA256ハッシュを表示します。

その他のプロセッサ

以下のプロセッサは、どのタブにも追加できます。

カスタム

カスタムの「エンコード/デコード」スクリプトは、スクリプトツリーで作成/追加し、カスタム出力パネルで使用できます。 それらはテンプレートに示されているように EncodeDecodeResult オブジェクトを返す必要があります。 しかし、 ユーザーがそうすることを怠った場合、それらは toString() 実装を活用して他のデータ型を返すことになります (これは一見すると役立つ/明らかではない場合があります)。

ユーティリティ

小文字に変換

入力をすべて小文字に変換します。

空白を削除

Character.isWhiteSpace(char) に基づいて、テキストからすべての空白文字を削除します。

反転

入力の順序を反転します。

大文字に変換

入力をすべて大文字に変換します。

ASCII化

互換モードでの動作により完全ではないかもしれませんが、アクセント/ダイアクリティカルマーク/合字を削除して、ASCII文字のみを残すようにテキストを変換します。 例:
  • Tĥïŝ ĩš â fůňķŷ Šťŕĭńġ: ﬁ. étrange. は This is a funky String: fi. etrange. になります。
  • 鸟儿 は空の文字列になります (すべての文字が削除されます)。

PowerShell エンコード

UTF-16LEに変換され、Base64エンコードされます。 「dir」をエンコードする以下の例のいずれかに相当します：

printf "dir"| iconv -t UTF-16LE | base64 -w 0 - 


$command = 'dir'
$bytes = [System.Text.Encoding]::Unicode.GetBytes($command)
$encodedCommand = [Convert]::ToBase64String($bytes)
echo $encodedCommand

結果: ZABpAHIA

これは、以下のようなものを通じて活用できます ($encodedCommand がエンコードされた値 ZABpAHIAに設定されている場合)：
powershell -exec bypass -enc $encodedCommand
