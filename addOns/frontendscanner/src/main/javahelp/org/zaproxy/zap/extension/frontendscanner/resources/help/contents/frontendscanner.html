Front-End Scanner

When installed and enabled, the Front-End Scanner add-on tampers with all HTTP responses passing through ZAP to inject a piece of JavaScript code containing custom logic, utility functions and scripts that ZAP users wish to run in the browser to help them audit front-end applications.

Add a script to be run in the browser

Note: as for now, only "Client-Side Passive" scripts are injected in web pages.

To add a passive script, go to the "Scripts" pane and right click on "Client-side Passive Rules", then "New Script...". Give your script a meaningful name, make sure that you use the template "scan.js" and save.

As they will be executed in a web browser, scripts need to be written in Javascript. For the same reason, they can make use of the Web APIs exposed by the browser that you use.

Each user script, written via ZAP's interface, ends up wrapped in a function in the front-end, receiving a Javascript object (FrontEndScanner) as a parameter. Thus, scripts have access to what this object exposes:

  • ZAP constants to help scripts create alerts,
  • the “mailbox”: a “publish-subscribe” mechanism to help scripts react to events happening in the browser (such as user interactions, Storage accesses, etc.),
  • a helper function to report findings back to ZAP.
All the functions containing user (passive) scripts will be executed automatically in the browser.

Enabling - Disabling

To toggle the add-on, go to "Tools" then "Toggle Front-End Scanner" (note that the add-on is turned off by default).

Note: Only enabled scripts will be injected in the targeted web pages. To enable (or disable) particular scripts, right click on them in the "Scripts" pane, then "Enable Script(s)" (or respectively "Disable Script(s)").
