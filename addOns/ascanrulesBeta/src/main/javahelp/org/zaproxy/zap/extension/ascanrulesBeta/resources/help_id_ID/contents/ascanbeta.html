Active Scan Rules - Beta

Berikut beta kualitas active scan aturan termasuk dalam hal ini add-on:

Backup File Disclosure

Scan untuk umum bernama salinan cadangan dari file pada web server, yang dapat mengungkapkan informasi sensitif.

Cross-Domain Misconfiguration

Memeriksa apakah web server dikonfigurasi untuk memungkinkan akses Domain Lintas, dari yang berbahaya layanan pihak ketiga, misalnya. Saat ini pemeriksaan untuk wildcard di Adobe crossdomain.xml, dan di SilverLight ini clientaccesspolicy.xml.

Token CSRF

Memindai keberadaan token anti-CSRF.
Peringatan atas permintaan yang tampaknya tidak mengandung token anti-CSRF.
Pos 2.5.0 kamu dapat menentukan daftar pengenal yang dipisahkan dengan tanda koma di parameter rules.csrf.ignorelist melalui panel 'konfigurasi aturan' pilihan. FORM mana pun dengan nama atau ID yang cocok dengan salah satu pengidentifikasi ini akan diabaikan saat memindai bukti kontra-CSRF yang hilang. Hanya gunakan fitur ini untuk mengabaikan FORMs yang anda tahu adalah aman, misalnya bentuk-bentuk pencarian.

Injeksi Ekspresi Bahasa

Periksa jika web aplikasi adalah subjek untuk penyerangan injeksi Ekspresi Bahasa (EL), yang terjadi ketika sebuah aplikasi gagal untuk menetralkan secara cukup elemen-elemen khusus yang dapat memodifikasi pernyataan EL dimaksudkan sebelum di jalankan.

Kerentanan membukaSSL

Mendeteksi apakah server web rentan terhadap kerentanan membukaSSL yang terkejut, dengan memanfaatkannya. Untuk keterangan lebih lanjut lihat CVE-2014-0160.

Parameter polusi HTTP (HPP)

Menyediakan duplikat atau banyak parameter HTTP dengan nama yang sama dapat menyebabkan aplikasi atau situs web untuk menafsirkan nilai dengan cara yang tidak disengaja. Dengan memanfaatkan efek ini, berbahaya individu mungkin bisa bypass validasi input, memicu kesalahan atau memodifikasi internal nilai-nilai variabel. Ada perbedaan perlakuan dari duplikat parameter berdampak baik klien (browser) dan server.

Metode HTTP tidak aman

Mendeteksi (dan eksploitasi, tergantung pada pengaturan pindaian) metode HTTP yang dikenal tidak aman yang diaktifkan untuk URL.

Integer Overflow Error

Terlihat untuk indikator integer overflow dalam menyusun kode yang menyebabkan web server crash. Itu tidak ini dengan menempatkan beberapa string bilangan bulat yang dirancang untuk mencoba dan merangsang buruk tanggapan.

Padding Oracle

Scanner ini upaya untuk memanipulasi padding dienkripsi string untuk memicu respon kesalahan menunjukkan kemungkinan padding oracle kerentanan. Kerentanan tersebut dapat mempengaruhi setiap aplikasi atau framework yang menggunakan enkripsi tidak benar, seperti beberapa versi ASP.net Java Server Faces, dan Mono.

Eksekusi Kode jarak Jauh - CVE-2012-1823

Mendeteksi CVE-2012-1823 untuk melakukan Eksekusi Kode jarak Jauh pada PHP-CGI web berbasis server.

Sesi Fiksasi

Sesi Fiksasi mungkin menjadi mungkin. Jika masalah ini terjadi dengan masuk URL (di mana pengguna mengotentikasi diri mereka sendiri untuk aplikasi), maka URL yang dapat diberikan oleh seorang penyerang, selain tetap session id, untuk korban, untuk kemudian mengasumsikan identitas korban menggunakan diberikan session id. Jika terjadi masalah dengan non-masuk halaman, URL dan tetap session id hanya dapat digunakan oleh penyerang untuk melacak aplikasi yang tidak berkepentingan tindakan pengguna. Jika terjadi kerentanan pada cookie bidang atau bentuk lapangan (POST parameter) bukan pada URL (GET) parameter, maka beberapa kerentanan juga mungkin diperlukan dalam rangka untuk mengatur cookie lapangan pada browser korban, untuk memungkinkan kerentanan yang dapat dimanfaatkan.

ShellShock - CVE-2014-6271

Scanner ini melakukan 2 serangan untuk mendeteksi server rentan terhadap CVE-2014-6271 alias ShellShock.
Yang pertama adalah sederhana tercermin serangan dan yang kedua adalah berdasarkan waktu serangan.

Posting 2.5.0 anda dapat mengubah panjang dari waktu yang digunakan untuk serangan dengan mengubah aturan.umum.tidur parameter melalui Pilihan 'Aturan konfigurasi' panel.

Pengungkapan sumber kode - CVE-2012-1823

Eksploitasi CVE-2012-1823 untuk mengungkapkan kode sumber PHP sisi server pada server web berbasis PHP-CGI.
Hanya analisis tanggapan yang berbasis teks (HTML, JS, JSON, XML, dll), untuk menghindari kesalahan positif yang mungkin terjadi pada gambar atau konten biner lainnya.

Pengungkapan sumber kode - SVN

Menggunakan metadata repositori kode sumber subversi untuk memindai berkas yang berisi kode sumber pada server web.

Pengungkapan sumber kode - /WEB-INF

Mengeksploitasi kehadiran folder / WEB-INF yang tidak dilindungi untuk mendownload dan mendekompilasi kelas Java, untuk mengungkapkan kode sumber Java.

SQL injeksi - Hipersonik (berbasis waktu)

Pemindai ini menggunakan sintaks SQL spesifik Hipersonik untuk mencoba menginduksi penundaan waktu dalam pernyataan SQL yang disebut oleh halaman.
Jika kueri yang tidak dimodifikasi tidak terpengaruh oleh penundaan waktu, dan penundaan kueri yang dimodifikasi dapat dikendalikan, ini adalah indikasi kerentanan SQL injeksi berbasis waktu dalam database SQL hipersonik.
Pemindai ini sensitif terhadap waktu, dan seharusnya hanya digunakan untuk menemukan SQL yang keras kepala dan tidak jelas kerentanan injeksi dalam database hipersonik yang dicurigai.
Untuk alasan ini, jumlah benang pemindai aktif harus diatur seminimal mungkin saat menggunakan pemindai ini, untuk meminimalkan beban pada server web, server aplikasi, dan database, untuk menghindari kesalahan positif yang disebabkan oleh penundaan beban bukan dengan penundaan injeksi SQL.
Tes pemindaian hanya untuk waktu dasar kerentanan injeksi SQL

Pos 2.5.0 Anda dapat mengubah jangka waktu yang digunakan untuk menyerang dengan mengubah rules.common.sleep parameter melalui pilihan panel 'konfigurasi pengaturan'.

Penghitungan nama pengguna

Ada kemungkinan untuk menghitung nama pengguna, berdasarkan tanggapan HTTP yang berbeda saat nama pengguna yang sah dan tidak sah disediakan. Ini akan sangat meningkatkan kemungkinan keberhasilan password brute-forcing serangan terhadap sistem. Catatan yang positif palsu kadang-kadang dapat diminimalisir dengan meningkatkan 'Attack Strength' Pilihan di ZAP. Silahkan periksa bidang 'Info Lainnya' secara manual untuk mengkonfirmasi jika ini sebenarnya sebuah masalah. Scanner ini dilewati jika tidak ada konteks yang didefinisikan yang menggunakan Otentikasi berbasis Bentuk, dan hanya berjalan melawan URL diidentifikasi sebagai masuk URL dari konteks.

Injeksi XPath

Seperti yang dijelaskan oleh OWASP: "Serangan injeksi XPath terjadi saat situs web menggunakan informasi yang dipasok pengguna untuk kueri XPath untuk data XML. Dengan mengirimkan informasi yang salah secara sengaja ke dalam situs web, penyerang bisa mengetahui bagaimana data XML disusun, atau mengakses data yang biasanya tidak bisa diakses. Malahan dia bisa meningkatkan keunggulannya pada situs web jika data XML digunakan untuk otentikasi (seperti berkas pengguna berbasis XML) atau otorisasi." Pemindai ini berusaha untuk mengidentifikasi kelemahan tersebut.

XXE

Komponen ini mencoba untuk mengidentifikasi aplikasi yang dapat XML Entitas eksternal (XXE) serangan. Aplikasi yang parse XML masukan yang mungkin dapat XXE ketika lemah atau buruk dikonfigurasi parsers menangani masukan XML yang berisi referensi untuk sebuah entitas eksternal seperti file lokal, permintaan HTTP untuk internal atau tersier sistem, dll.

Hal ini membutuhkan Callback ekstensi, sehingga tidak akan bekerja jika ekstensi ini dinonaktifkan atau dihapus. Hal ini juga dianjurkan bahwa anda menguji bahwa Callback ekstensi adalah dikonfigurasi dengan benar untuk situs target. Jika sistem target tidak dapat terhubung ke Callback Alamat kemudian beberapa XXE kerentanan tidak akan terdeteksi.
