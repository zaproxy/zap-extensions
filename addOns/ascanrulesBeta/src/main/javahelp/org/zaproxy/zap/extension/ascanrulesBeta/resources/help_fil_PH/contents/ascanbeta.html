Ang aktibong pag-iscan ng mga patakaran - ng beta

ang sumusunod na kalidad ng beta ng aktibong pag-iskan ay isinama sa add-on na ito:

ang backup na payl ng pagbubunyag

ang mga pag-iskan para sa karaniwang-ipinangalan ng mga kopya ng backup ng mga payl sa mga server ng web, kung saan ay maaaring ihayag ang sensitibong impormasyon.

ang cross-dominyo ng misconfiguration

ang pagsisiyasat kung ang web ng server ay nakonpiguraupang pahintulutan ang cross na dominyo ng pagpapatakbo, mula sa isang malisyosong serbisyo ng ikatlong partido, para sa halimbawa. Kasalukuyang pagsisiyasat para sa mga wildcard sa mga adobe ng crossdomain.xml,ang sa mga silverlight ng clientaccesspolicy.xml.

CSRF Token

Sinusuri ang pagkakaroon ng mga token ng Anti-CSRF.
Mga Alerto sa mga kahilingan na hindi lilitaw na naglalaman ng mga token ng Anti-CSRF.
I-post ang 2.5.0 maaari mong tukuyin ang isang listahan ng pinaghiwalay ng kuwit ng mga identifier sa Parameter na rules.csrf.ignorelist sa pamamagitan ng panel ng pagsasaayos ng 'Rule configuration'. Ang anumang FORM na may pangalan o ID na tumutugma sa isa sa mga tagatukoy na ito ay hindi papansinin kapag nag-scan para sa mga nawawalang mga token ng Anti-CSRF. Gamitin lamang ang feature na ito para huwag pansinin ang FORMs na malalaman mo na ligtas, halimbawa pagsusuri ng mga anyo.

Expression Language Injection

Suriin kung ang aplikasyon sa web ay paksa sa Expression Language (EL) inaatake ng iniksyon, ang nangyayari kapag ang isang application ay nabigo sa sapat na neutralisahin ang mga espesyal na elemento na maaaring baguhin ang inilaan EL pahayag bago ito isagawa.

Heartbleed OpenSSL Vulnerability

Nakita kung ang web server ay mahina sa Heartbleed OpenSSL Vulnerability, sa pamamagitan ng paggamit nito. Para sa mga karagdagang detalye sumangguni sa CVE-2014-0160.

HTTP Parameter Pollution (HPP)

Ang pagbibigay ng duplicate o maraming mga parameter ng HTTP na may parehong pangalan ay maaaring maging sanhi ng isang application o website upang bigyang-kahulugan ang mga halaga sa mga hindi nilalayong paraan. Sa pamamagitan ng paggamit ng mga epekto na ito, isang nakakahamak na indibidwal maaaring ma-bypass ang pagpapatunay ng pag-input, mai-trigger ang mga error o baguhin ang mga panloob na halaga ng variable. Mayroong pagkakaiba sa paggamot ng mga duplicate na parameter na nakakaapekto sa parehong mga kliyente (mga browser) at mga server.

Insecure HTTP Method

Nakikita (at nagsasamantala, depende sa mga setting ng pag-scan) na kilala ang mga pamamaraan ng hindi ligtas na HTTP na pinagana para sa URL.

Integer Overflow Error

Ang hitsura para sa mga tagapagpahiwatig ng overflows ng integer sa naipon na code na nagiging sanhi ng crash ng web server. Ginagawa nito ito sa pamamagitan ng paglagay ng maraming mga string ng integers na dinisenyo upang subukan at pasiglahin ang mga masamang tugon.

Padding Oracle

Sinisikap ng scanner na manipulahin ang padding ng naka-encrypt na mga string upang ma-trigger ang isang tugon ng error na nagpapahiwatig ng malamang na paglalagay ng kahinaan sa orakulo. Ang ganitong kahinaan ay maaaring makaapekto sa anumang aplikasyon o balangkas na gumagamit ng hindi wastong pag-encrypt, tulad ng ilang bersyon ng ASP.net, Java Server Faces, at Mono.

Remote Code Execution - CVE-2012-1823

Tiktikan ang CVE-2012-1823 upang maisagawa ang Remote Code Execution sa isang PHP-CGI na web server na batay.

Session Fixation

Maaaring posible ang pag-aayos ng Session. Kung ang isyu na ito ay nangyayari sa isang login na URL (kung saan pinatutunayan ng user ang kanilang sarili sa application), maaaring maibigay ang URL ng isang magsasalakay, kasama ang isang fixed session id, sa isang biktima, upang ipagpalagay na mamaya ang pagkakakilanlan ng biktima gamit ang ibinigay na session id. Kung ang isyu ay nangyayari na may isang hindi naka-login na pahina, ang URL at fixed session id ay maaari lamang gamitin ng isang magsasalakay upang masubaybayan ang mga aksyon na hindi awtorisadong user. Kung ang kahinaan ay nangyayari sa isang patlang ng cookie o isang larangan ng anyo (POST parameter) sa halip na sa isang URL (GET) na parameter, pagkatapos ay ang ilan pang mga kahinaan ay maaaring kinakailangan din upang itakda ang larangan ng cookie sa browser ng biktima, upang payagan ang kahinaan upang mapagsamantalahan.

ShellShock - CVE-2014-6271

Ang scanner na ito ay nagsasagawa ng 2 atake upang makita ang mga server na mahina sa CVE-2014-6271 o ShellShock.
Ang una ay isang simpleng sumasalamin na pag-atake at ang pangalawa ay isang pag-atake batay sa oras.

Mag-post ng 2.5.0 maaari mong baguhin ang haba ng oras na ginamit para sa pag-atake sa pamamagitan ng pagbabago ng parameter na rules.common.sleep sa pamamagitan ng panel ng pagsasaayos ng Rule ng Mga Pagpipilian.

Pagbubunyag ng Code ng Pinagmulan - CVE-2012-1823

Samantalahin ang CVE-2012-1823 upang ibunyag ang server ng pinagmulan ng PHP source code sa isang web server na PHP-CGI batay
Pinag-aaralan lamang ang mga tugon na batay sa teksto (HTML, JS, JSON, XML, atbp), upang maiwasan ang mga maling positibo na maaaring mangyari sa imahe o iba pang nilalaman ng binary.

Source Code Disclosure - SVN

Gumagamit ng Pagbabagsak ng pinagmulan ng imbakan ng code metadata para i-scan sa mga file na naglalaman ng pinagmulan ng code na nasa web server.

Source Code Disclosure - /WEB-INF

Isabuhay ang pagkakaroon ng isang walang protektadong / WEB-INF na folder upang i-download at i-decompile ang mga klase sa Java, upang ibunyag ang pinagmulan ng Java code.

SQL Injection - Hypersonic (Time Based)

Ang scanner na ito ay gumagamit ng Hypersonic-specific SQL syntax upang subukang hulihin ang mga pagkaantala sa oras sa pahayag ng SQL na tinatawag sa pamamagitan ng pahina.
Kung ang hindi nabagong tanong ay hindi naapektuhan ng pagka-antala ng oras, at maaaring kontrolin ang pagkaantala ng nabago na tanong, ito ay nagpapahiwatig ng isang oras na nakabatay sa SQL Injection kahinaan sa isang Hypersonic SQL database.
Ang scanner na ito ay sensitibo sa oras, at dapat lamang gamitin sa isang pagtatangka upang makahanap ng pag mama-tigas at di-halatang SQL mga kahinaan sa iniksyon sa isang pinaghihinalaang Hypersonic database.
Para sa kadahilanang ito, ang bilang ng mga aktibong i-scan ang mga thread ay dapat itakda sa pinakamababa kapag ginagamit ang scanner na ito, upang mabawasan ang pag-load sa web server, application server, at database, upang maiwasan ang mga maling positibo na sanhi ng load pagkaantala sa halip na sa pamamagitan ng SQL pagkaantala pagkaantala.
Ang mga pagsusulit ng scanner lamang para sa mga oras na nakabatay sa SQL injection na mga kahinaan.

Mag-post ng 2.5.0 maaari mong baguhin ang haba ng oras na ginamit para sa pag-atake sa pamamagitan ng pagbabago ng parameter narules.common.sleepsa pamamagitan ng panel ng pagsasaayos ng 'Rule configuration'.

Username Enumeration

Maaaring posible na magbilang ng mga username, batay sa magkakaibang mga tugon ng HTTP kapag ang wasto at hindi wastong mga pangalan ng gumagamit ay ibinigay. Ito ay lubos na mapapalaki ang posibilidad ng tagumpay ng password na pag-atake ng brute laban sa sistema. Tandaan na ang mga maling mga positibo ay maaaring paminsan-minsang mapaliit sa pamamagitan ng pagtaas ng Pagpipilian sa 'Pag-atake ng Lakas' sa ZAP. Mangyaring manu-manong suriin ang patlang ng 'Iba pang Impormasyon' upang kumpirmahin kung ito talaga ang isyu. Ang scanner na ito ay nilaktawan kung walang mga konteksto na tinukoy na paggamit ng Form-based na Pagpapatunay, at nagpapatakbo lamang laban sa URL na kinilala bilang ang login na URL ng isang konteksto.

XPath Injection

Tulad ng inilarawan ng OWASP: "Ang pag-atake ng XPath Injection ay nangyayari kapag ang isang web site ay gumagamit ng impormasyon na ibinigay ng user bumuo ng isang XPath query para sa XML data. Sa pamamagitan ng pagpapadala ng sinasadyang maling impormasyon sa web site, maaaring tuklasin ng isang magsasalakay kung paano nakaayos ang data ng XML, o ma-access ang data na maaaring hindi niya normal na ma-access sa. Maaari pa rin niyang itaas ang kanyang mga pribilehiyo sa web site kung ginagamit ang data ng XML authentication (tulad ng isang XML na gumagamit ng user na file) o awtorisasyon. "Sinisikap ng scanner na ito na makilala ang naturang mga kahinaan.

XXE

Sinisikap ng sangkap na ito na makilala ang mga application na napapailalim sa pag-atake ng XML eXternal Entity (XXE). Ang mga application na ma-parse ang input ng XML ay maaaring napapailalim sa XXE kapag mahina o hindi maayos na naka-ayos ang parser hawakan ang input ng XML na naglalaman ng sanggunian sa isang panlabas na entidad tulad ng isang lokal na file, mga kahilingan ng HTTP panloob o tertiary system, atbp.

Nangangailangan ito ng extension ng Balik ng tawag, kaya hindi gagana kung ang extension na ito ay hindi pinagana o inalis. Inirerekomenda rin na subukan mo na ang extension ng Callback ay wastong na-configure para sa iyong target na site. Kung ang sistema ng target ay hindi maaaring kumonekta sa Addressback Address pagkatapos ang ilang mga XXE na mga kahinaan ay hindi napansin.
