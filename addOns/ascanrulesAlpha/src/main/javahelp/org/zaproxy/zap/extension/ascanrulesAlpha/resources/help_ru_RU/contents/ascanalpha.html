Внедрение LDAP

Возможно внедрение LDAP. Злоумышленник может обойти элементы управления аутентификацией, а также просмотреть и изменить произвольные данные в каталоге LDAP.

Последний код: LdapInjectionScanRule.java

Внедрение NoSQL - MongoDB

Это правило пытается идентифицировать уязвимости, связанные с внедрением NoSQL в MongoDB. Он пытается предпринять различные типы атак, в том числе: логические, основанные на ошибках, основанные на времени и обход аутентификации. Он также будет пытаться использовать полезные данные, специфичные для параметра JSON, если сканирование настроено на включение вариантов параметров JSON. Последний код: MongoDbInjectionScanRule.java

Заголовок CORS

Это правило пытается идентифицировать заголовки CORS, а также неправильную конфигурацию CORS. CORS считается неправильно настроенным, если он разрешает все источники, источники с более слабыми протоколами и нулевое происхождение.

Последний код: CorsScanRule.java

Обман веб-кеша

Это правило пытается идентифицировать уязвимости Web Cache Deception. Он проверяет, можно ли использовать статический путь, добавленный к исходным URI, для утечки конфиденциальной информации пользователя.
  • Пользователь должен пройти аутентификацию перед использованием этого правила
Последний код: WebCacheDeceptionScanRule.java

Актуаторы Java Spring

Это правило пытается определить, включены ли пружинные приводы. Тесты для маршрута по умолчанию / исполнительный механизм / работоспособность в приложении. SpringActuatorScanRule.java

Log4Shell (CVE-2021-44228)

Это правило пытается обнаружить Log4Shell ( CVE-2021-44228) уязвимость. Он полагается на надстройку OAST для генерации внеполосных данных и проверки взаимодействия с DNS. Мы рекомендуем использовать это правило сканирования с включенной инъекцией заголовка для максимального охвата. Последний код: Log4ShellScanRule.java
