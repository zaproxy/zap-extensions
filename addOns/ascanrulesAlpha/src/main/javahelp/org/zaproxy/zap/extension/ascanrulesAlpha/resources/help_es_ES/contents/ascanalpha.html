Reglas de Análisis Activo - alpha

Las siguientes reglas de análisis activo de calidad alfa están incluidas en este complemento:

Encabezado Dos de Apache Range (CVE-2011-3192)

Comprueba si el servidor es vulnerable al problema de denegación de servicio del encabezado de rango Apache, solicitando once (11) diferentes rangos de bytes. Once rangos es uno más de lo aceptado por los servidores parcheados / fijos.

Detector de cookie flojo

Evalúa los cookies para detectar si alguno no tiene efecto en tamaño receptivo cuando se omite, especialmente los cookie que contienen el nombre "session" o "userid"

Escaner de ELMAH

Evalúa si los módulos de error de registro y los manejadores (elmah.axd) del módulo HTTP están disponibles. A pesar de que este módulo es util para desarrolladores y otros interesados, este también puede filtrar una cantidad importante de información que algún analista de seguridad o individuo malicioso podría estar interesado.

El escaner ELMAH tiene como objetivo tecnologías basadas en Microsoft: IIS, Windows, ASP, y MSSQL.

Ejemplo archivo activo explorador

Esto implementa una regla de exploración activa de ejemplo que carga cadenas de un archivo que el usuario puede editar.
Para más detalles ven: http://zaproxy.blogspot.co.uk/2014/04/hacking-zap-4-active-scan-rules.html

Scanner activo Simple ejemplo

Esto implementa una regla de exploración activa de ejemplo muy simple.
Para más detalles ven: http://zaproxy.blogspot.co.uk/2014/04/hacking-zap-4-active-scan-rules.html

Expresión lenguaje inyección

El software construye todo o parte de una instrucción de lenguaje (EL) de expresión un Java Server Page (JSP) con entrada influenciado externamente de un componente ascendente, pero este no neutralizar o neutraliza incorrectamente elementos especiales que podrían modificar la instrucción del "EL" previsto antes de ejecutarse. En ciertas versiones de Spring 3.0.5 y anteriores, había una vulnerabilidad (CVE-2011-2730) en la que la expresión del lenguaje de las etiquetas podía ser evaluado dos veces, lo que efectivamente exponía cualquier aplicación a la inyección de EL. Sin embargo, aun en versiones posteriores, esta debilidad is aun posible dependiendo de la configuración.

GET for POST Scanner

This scanner takes application/x-www-form-urlencoded POST requests, changes the parameters from POST to GET and resubmits the request. Si la respuesta GET es la misma que la respuesta POST original aparece una alerta. A pesar de que esto no necesariamente representa una debilidad de seguridad en sí mismo, puede indicar que otros ataques o debilidades pueden expandirse o simplificarse. (Como un POST basado en un ataque Cross-Site Scripting (XSS) que cambie a GET.)

HTTP Only Site

Este escáner activo comprueba si un sitio HTTP sirve bajo HTTPS.

Httpoxy

Este escáner activo comprueba si un sitio está utilizando el encabezado HTTP Proxy especificado en la solicitud.
Configura un proxy HTTP el cual escucha todas las interfaces de un puerto libre asignado aleatoriamente. Luego envía una serie de solicitudes al servidor destino con el encabezado Proxy HTTP configurado a cada uno de las direcciones de IP disponibles y el puerto que está siendo escuchado. Si una solicitud es recibida en el nuevo puerto, entonces el posible que el servidor sea vulnerable.
IMPORTANTE - la computadora donde se ejecuta ZAP debe aceptar las solicitudes entrantes en puertos arbitrarios. Si un firewall evita conexiones entrantes esta regla no funcionará.

HTTPS como un escáner HTTP

Este escáner activo intenta acceder al contenido que originalmente fue accedido a través de HTTPS (SSL/TLS) mediante HTTP.

Inyección LDAP

La inyección LDAP puede ser posible. Puede ser posible para un atacante eludir los controles de autenticación, y ver y modificar datos arbitrarios en el directorio LDAP.

Inyección SQL - MsSQL

Este escáner activo intenta inyectar comandos MsSQL específicos de suspensión en valores de los parámetros y analiza el tiempo de respuesta del servidor para ver si la suspensión se ejecuto con éxito en el servidor (indicando un exitoso ataque de inyección SQL).

Revelación del código fuente - Inclusión de archivo

Utiliza las técnicas de inclusión de archivo local para analizar archivos que contengan el código fuente en el servidor de la web.

Revelación del código fuente - Git

Utiliza los metadatos del repositorio del código fuente de Git para analizar archivos que contengan el código fuente en el servidor web.

Escáner de Trace.axd

Comprueba si el visor Trace (trace.axd) está disponible. A pesar de que este componente sea conveniente para los desarrolladores y otros interesados, puede filtrar una cantidad importante de información, la cual puede interesarle a un analista de seguridad o a un individuo malicioso.

El escáner trace.axd tiene como objetivo las tecnologías basadas en Microsof: IIS, Windows, ASP, y MSSQL.

User Agent Fuzzer

Este escáner comprueba las diferencias de respuesta basado en agente de usuario inexacto (por ejemplo. sitios móviles, acceso como un rastreador de motor de búsqueda). Este escáner compara la respuesta de statuscode y el hascode del cuerpo de la respuesta con la respuesta original.
