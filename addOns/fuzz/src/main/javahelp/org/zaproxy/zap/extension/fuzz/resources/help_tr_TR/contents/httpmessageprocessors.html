HTTP Mesaj İşlemcileri

Mesaj İşlemcileri fuzz'lanmakta olan mesajlara erişebilir ve onları değiştirebilir, fuzzing sürecini kontrol edebilir ve ZAP UI'ı ile etkileşime geçebilir.

Yerleşik HTTP Mesaj İşlemcileri'nde bulunanlar:

Anti-CSRF Token Yenileyici

Talepte bulunan anti-CSRF token'larını yenilemeye yarar. Bu işlemcinin kullanılabilmesi için anti-CSRF token'larının ZAP tarafından düzgünce algılanması gerekmektedir.
Daha fazla bilgi için "Başlarken" > "Öne Çıkanlar" > "Anti CSRF Token'ları" yardım sayfasına danışın.
Not: Eğer anti-CSRF token'ları algılanırsa, bu işlemci otomatik olarak işlemciler listesine eklenir.

Fuzzer HTTP İşemcisi (Script)

Etkinleştirilmiş Fuzzer HTTP İşlemcisi script'lerini seçmeye yarar. Bu script'ler şunları yapmanızı sağlar:
  • Payload'ların listesini edinme
  • Fuzzing'i durdurma
  • Hata sayımını arttır
  • Yeni mesajlar gönder
  • Sonuçlar sekmesine yeni mesajlar ekle
  • Fuzzer sekmesine özel 'durum' mesajları ayarı
  • Uyarıları yükselt

Payload Yansıma Algılayıcısı

" kullanarak Durum sütünunda, yanıtta enjekte edilen payload'lardn birinin bulunup bulunmadığını belirtir. Yansıtılmış".
Not: İşlemci işlemcilerin listesine otomatik olarak eklenir.

İstek İçerik-Uzunluğu Güncelleyici

Günceller (ya da zaten yoksa ekler) İçerik-Uzunluğu talep gövdesinin uzunluğu ile tüm talep yöntemleri için başlığı. Talep gövdesinin boyutu sıfırsa ve başlık mevcut değilse herhangi bir değişiklik yapılmaz.
Not: İşlemci işlemcilerin listesine otomatik olarak eklenir.

Etiket Yaratıcı

Sonuçlar tablosunun Durum sütununa yanıtın içeriğine bağlı olarak özel 'etiketler' eklemeye yarar

Kullanıcı Mesaj İşlemcisi

Fuzz'lanmakta olan HTTP mesajını içeren bağlamlarda tanımlanmış kullanıcılardan birini kullanarak, bir kullanıcı olarak fuzz'lamayı sağlar.

Diğer eklentiler ek HTTP Mesaj İşlemcileri tanımlayabilir.
