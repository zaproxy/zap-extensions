Параметры Экран приложений

Этот экран позволяет настроить Приложения тот
По умолчанию нет доступных приложений, нужно добавить все приложения, которые вы хотите использовать.

Показать имя

Имя, которое будет использоваться для этого приложения в ZAP.

Полная Команда

Полный путь к приложению, которое вы хотите вызвать.
Можно указать только одну команду. Если вы хотите запустить несколько команд в одном go ( пути) или настроить переменные окружения, то вы должны создать сценарий или командный файл и затем вызовите это из ZAP.

Параметры

Параметры, которые ZAP должен передать команде.
Для передачи параметров могут использоваться следующие теги:
     %url% полный URL, например 'http://localhost/test?a=b'
     %site% сайт, например 'http://локальный:8080/'
     %host% имя хоста, например. «локальный хост» ('localhost')
     %port% порт, напр. '80'
     %cookie% первое поле cookie из заголовка запроса (если есть)
     %postdata% отправленные данные POST, если таковые имеются, с заменой любых новых строк на "\n"
     %msgid% идентификатор HTTP-сообщения для получения данных из API (/JSON/core/view/message/?id=$msgid), не всегда существует, вернет -1, если нет идентификатора сообщения
     %header-{{ header }}% заголовок запроса по имени (если есть), например. %header-user-agent% возвращает значение заголовка User-Agent.
Таким образом, если команда принимает URL-адрес с использованием «-u», вы должны указать параметры как: «-u %url%»

Захват вывода

Если этот флажок установлен, запуск команды и любой вывод, который она генерирует, будет отображаться на Вкладка «Вывод» .
Вы всегда должны использовать эту опцию при запуске скриптов или команд, которые не имеют пользовательский интерфейс.
Вы также можете использовать эту опцию для устранения неполадок — если приложение не запустите, как ожидалось, затем скопируйте команду run и попробуйте ее из командной строки.

Вывод в заметку

Если этот флажок установлен, любой вывод будет добавлен к Примечание прилагается к соответствующему Вкладка История запись
Обратите внимание, что некоторые узлы в Закладка сайты не имеют связанных Вкладка История записи - для этих узлов эта опция не будет действовать.
