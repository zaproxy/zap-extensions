WebSockets

Las WebSockets pueden utilizarse por aplicaciones web o sitios web para configurar un canal bidireccional (dos vías) de comunicación full duplex sobre una sola conexión TCP.
Cuenta con un protocolo liviano que permite a los desarrolladores percatarse de los casos de uso en tiempo real. Las WebSockets también proporcionan una alternativa para el uso intensivo de Ajax, HTTP Long Polling o Comet.

Después de un protocolo de enlace inicial basado en HTTP, la conexión TCP se mantiene abierta, permitiendo a las aplicaciones enviar y recibir datos arbitrarios. Por menudo se utiliza el puerto 80 el 443 para los canales Websocket encriptados.

El WebSocket estándar se define en

La API WebSocket (http://www.w3.org/TR/websockets/)
especifica la interfaz en navegadores
El protocolo WebSocket (RFC6455) (https://tools.ietf.org/html/rfc6455)
describe la estructura de los marcos sobre TCP

ZAP es capaz de:

  • interceptar y mostrar mensajes de WebSocket
  • establecer puntos de interrupción en los tipos específicos de mensajes de WebSocket
  • confunde los mensajes WebSocket (envía muchos datos inválidos o inesperados al navegador o servidor)
  • passively scan WebSocket messages and raise alerts with scripts
Los mensajes WebSocket se muestran dentro de la pestaña WebSockets.

The WebSocket add-on adds new scripts and additional endpoints to the ZAP API
