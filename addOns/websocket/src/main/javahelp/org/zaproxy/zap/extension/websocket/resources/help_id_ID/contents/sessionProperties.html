Properti sesi khusus Websocket

pengecualian dari Websocket

Ini memungkinkan anda untuk mengatur URL dimana komunikasi Websocket tidak diproses dalam ZAP.
Meskipun semua pesan dari URL yang dikecualikan diteruskan, tidak ada yang disimpan tidak dapat anda melihat mereka antarmuka pengguna. Fitur ini berguna untuk volume tinggi, performa kritis koneksi Websocket.
Anda dapat membedakan pada port untuk satu domain yang diberikan, dengan menggunakan contohnya: example.com:443 untuk mengecualikan koneksi WebSocket dari example.com saja pada port 443.
