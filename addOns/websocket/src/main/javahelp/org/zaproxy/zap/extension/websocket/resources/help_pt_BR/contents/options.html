Opções específicas de WebSocket

Esta tela permite que você configure as opções de WebSockets:

Encaminhar tudo

Se você não está interessado na comunicação via WebSockets, mas deseja permitir que essas mensagens trafeguem pelo ZAP, deve habilitar esta opção. Como resultado nenhuma mensagem será armazenada no banco de dados de sessão. Além disso, nenhuma mensagem aparecerá na guia de WebSockets. Essa configuração pode ser útil se você tem que lidar com conexões de WebSocket críticas em desempenho, e não se interessa pelo que está sendo enviado.

Se você quiser ignorar um canal específico, confira as Propriedades de Sessão específicas de Websocket.

Parar tudo

Na barra de Interrupção existem dois botões:
  •   Definir interrupção em todas as requests
  •   Definir interrupção em todas as respostas
Quando você habilita essa opção, todas as mensagens de WebSocket que saem (e respectivamente as que entram) também são detectadas por esses botões de controle.

Por padrão, esta opção é desabilitada. Neste caso, apenas as mensagens HTTP são retidas, mas você pode adicionar pontos de interrupção explícitos sobre WebSockets comunicação na Aba de WebSocket.

Interrupção nas mensagens de Ping e Pong

PING e PONG são mensagens muitas vezes usadas pelos servidores de WebSocket para determinar a saúde da conexão. Se você pegar uma mensagem PING entrando, o navegador não é capaz de retornar um PONG imediatamente, resultando em um valor de latência ruim.

Se esta opção estiver desativada nenhuma mensagem de PING ou PONG é capturada quando você:
  • definir um ponto de interrupção em todas as solicitações ou todas as respostas, ou
  • seguir para o próximo request ou resposta e uma mensagem de PING ou PONG chega
Essa configuração não afeta os pontos de interrupção são definidos explicitamente para PING ou PONG mensagens. Você pode exibir esses pontos de interrupção explícitos na Aba dos pontos de interrupção.

Remover cabeçalho Sec-WebSocket-Extensions

Permite remover o cabeçalho Sec-WebSocket-Extensions das mensagens de handshake, assim nenhuma transformação é feita às mensagens WebSocket enviadas/recebidas.
This option should always be enabled unless the client or the server under test requires them. The WebSocket messages might not be correctly processed by ZAP when extensions are used.
