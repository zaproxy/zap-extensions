WebSockets

WebSockets можно использовать веб-приложениями или веб-сайтами для настройки двунаправленные (двусторонние), полный канал дуплексного связи над Одиночное соединение TCP.
Он имеет легкий протокол, позволяющий разработчикам реализовать случаи использования в реальном времени. Websockets также предоставляют альтернативу Тяжелое использование AJAX, HTTP длинного опроса или кометы.

После первоначального рукопожатия на основе HTTP соединение TCP остается открытым, разрешение приложений для отправки & AMP; получать произвольные данные. Часто порт 80 или 443 для зашифрованных каналов WebSocket используются.

The WebSocket standard is defined in

The WebSocket API (http://www.w3.org/TR/websockets/)
Определяет интерфейс в браузерах
The WebSocket Protocol (RFC6455) (https://tools.ietf.org/html/rfc6455)
Описывает структуру рамок Websocket на TCP

ZAP может:

  • Перехватить и показать сообщения WebSocket
  • Установить точки останова на определенные типы сообщений Websocket
  • fuzz Websocket cообщения (отправьте много недействительных или неожиданных данных в браузер или сервер)
  • Пассивно сканировать сообщения WebSocket и повышать оповещения со скриптами
Сообщения Websocket отображаются в пределах Вкладки WebSockets .

Настройка WebSocket добавляет новый Сценарии и дополнительные конечные точки на ZAP API
