WebSockets

Websockets dapat digunakan oleh aplikasi web atau situs web untuk menyiapkan bi-diretional (dua-arah), saluran lengkap komunikasi duplex melalui koneksi TCP tunggal.
Dilengkapi dengan protokol ringan, memungkinkan pengembang untuk menyadari kasus penggunaan sebenarnya. WebSockets juga memberikan alternatif untuk berat penggunaan Ajax, HTTP Panjang Polling atau Komet.

Setelah awal berbasis HTTP jabat tangan, koneksi TCP tetap terbuka, memungkinkan aplikasi untuk mengirim & menerima data yang sewenang-wenang. Sering pelabuhan 80 atau 443 untuk dienkripsi WebSocket saluran yang digunakan.

Dengan WebSocket standar didefinisikan dalam

The WebSocket API (http://www.w3.org/TR/websockets/)
menentukan interface dalam browser
The WebSocket Protocol (RFC6455) (https://tools.ietf.org/html/rfc6455)
menggambarkan struktur bingkai Websocket pada TCP

ZAP mampu untuk:

  • mencegat dan menampilkan pesan Websocket
  • mengatur breakpoint pada jenis tertentu pesan Websocket
  • mengaburkan pesan Websocket (Mengirim banyak data tidak valid atau tidak terduga ke browser atau server)
  • passively scan WebSocket messages and raise alerts with scripts
Pesan Websocket ditampilkan dalam Tab Websocket.

The WebSocket add-on adds new scripts and additional endpoints to the ZAP API
