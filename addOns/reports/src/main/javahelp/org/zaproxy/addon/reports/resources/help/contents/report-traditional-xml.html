Script Diagnostics Section

When the scriptdiagnostics section is enabled and the Script Console add-on has recorded runs in the session database, the report can include a scriptDiagnostics element containing run elements with script diagnostic records (for example failed automation runs). For example: 
        <scriptDiagnostics>
                <run>
                        <created>2026-05-13T12:19:50.536360Z</created>
                        <outcome>FAILED</outcome>
                        <summary>Job: script Chain order: account_check -> nav failed.</summary>
                        <scripts>
                                <script>
                                        <order>1</order>
                                        <scriptName>account_check</scriptName>
                                        <scriptType>standalone</scriptType>
                                        <steps>
                                        </steps>
                                </script>
                                <script>
                                        <order>2</order>
                                        <scriptName>nav</scriptName>
                                        <scriptType>standalone</scriptType>
                                        <steps>
                                                <step>
                                                        <sourceStepIndex>13</sourceStepIndex>
                                                        <line>ZestClientElementClick</line>
                                                        <outputs>
                                                                <output>
                                                                        <kind>ERROR</kind>
                                                                        <message>ZestClientElementClick - Expected condition failed...</message>
                                                                </output>
                                                        </outputs>
                                                </step>
                                        </steps>
                                </script>
                        </scripts>
                </run>
        </scriptDiagnostics>
