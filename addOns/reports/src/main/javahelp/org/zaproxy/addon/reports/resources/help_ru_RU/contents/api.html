API генерации отчетов

В API добавлены следующие операции:

Просмотры

  • шаблоны: просмотр доступных шаблонов.
  • templateDetails (шаблон*): просмотр сведений об указанном шаблон.

Действия

  • генерировать (название* шаблон* тема описание контексты сайты включены разделыКонфиденциальность включенаОтчет о рискахИмя файла reportFileNamePattern reportDir display): создание отчета с поставляемые параметры.
    • заглавие: Название отчета
    • шаблон: Шаблон отчета
    • тема: Тема доклада
    • описание: Описание отчета
    • contexts: имя контекстов, которые будут включены в отчет, разделенный '|'. Например, «Контекст по умолчанию|Мой Контекст».
    • сайты: URL-адреса сайтов, которые должны быть включены в отчет, разделены символом '|'.
    • разделы: разделы отчета, которые должны быть включены, разделены символом '|'. Просмотр названий разделов через представление API templateDetails (шаблон*)
    • includeConfidences: Доверительные отношения, которые должны быть включены в отчет, разделенный '|'. Допустимые значения: «Ложноположительные», «Низкий», «Средний», «Высокий» и «Подтвержденный».
    • включенные риски: риски, которые должны быть включены в отчет, разделены символом '|'. Допустимые значения: «Информационное», «Низкое», «Средний» и «Высокий».
    • reportFileName: имя файла сгенерированного отчета. Это значение переопределяет параметр reportFileNamePattern.
    • reportFileNamePattern: шаблон имени файла отчета. например, {{гггг-мм-дд}}-ZAP-отчет-[[сайт]].
    • reportDir: Путь к каталогу, в котором находится сгенерированный отчет должны быть размещены.
    • display: отображение сгенерированного отчета. Либо "правда", либо "ложный".
