Script Diagnostics Section

When the scriptdiagnostics section is enabled and the Script Console add-on has recorded runs in the session database, the report can include a scriptDiagnostics object containing a runs array of script diagnostic records. Each run has created , outcome (for example FAILED or SUCCESS ), summary , and a scripts array. 
	"scriptDiagnostics": {
		"runs": [
			{
				"created": "2026-05-13T12:19:50.536360Z",
				"outcome": "FAILED",
				"summary": "Job: script Chain order: account_check -> nav failed.",
				"scripts": [
					{ â¦ }
				]
			}
		]
	}
Each script in a run has an order, name, type, and steps array. Failure steps include the source statement index, line (element type), and outputs (for example error messages). 
	{
		"sourceStepIndex": 13,
		"line": "ZestClientElementClick",
		"outputs": [
			{
				"kind": "ERROR",
				"message": "â¦"
			}
		]
	}
More data can be included on steps depending on the additional diagnostics sections enabled for the report, as outlined below.

Output for Script Diagnostics

Steps can include print output lines with kind OUTPUT . For example: 
	{
		"sourceStepIndex": 2,
		"line": "ZestActionPrint",
		"outputs": [
			{
				"kind": "OUTPUT",
				"message": "In example_script"
			}
		]
	}
When this section is disabled, steps still include ERROR output lines; only OUTPUT lines are omitted.
