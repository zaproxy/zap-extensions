повторное посещение (Revisit)

Вы когда-нибудь хотели иметь возможность просматривать приложение, которое вы тестировали с помощью ZAP, в определенное время? возможно, когда у вас нет к нему доступа или после того, как он каким-то образом изменился?
Если да, то это дополнение для вас!

Надстройка позволяет указать ZAP возвращать содержимое из истории ZAP между указанными даты, а не пересылать запросы на сайт.
Это позволяет вам явно просматривать и использовать веб-сайт, к которому у вас больше нет доступа.
Это также позволяет вам увидеть, как это выглядело между определенными моментами времени, что полезно, если с тех пор приложение изменилось.

При возврате страниц из истории ZAP плавающие нижние колонтитулы внедряются в HTML-страницы для дать визуальное представление о том, что эти страницы созданы с помощью ZAP.
ZAP будет возвращать страницы, полученные в результате запросов GET и POST, если найдет совпадающие.
Страницы совпадают, только если все параметры URL и POST совпадают, игнорируя порядок и любые известные токены CSRF.
Если в истории не удается найти подходящую страницу, возвращается пользовательская страница 404, которая будет включать ссылки на ту же страницу, если она была вызвана с другими параметрами или вне указанного периода времени.

Это дополнение добавляет контекстное меню в дерево сайтов:

Включить повторное посещение

При выборе будет показано всплывающее окно, в котором запрашивается время начала и окончания.
Время начала по умолчанию равно времени первого запроса в сеансе.
Время окончания по умолчанию равно времени последнего запроса в сеансе.
Если какие-либо элементы времени (например, «день месяца») не изменились, эти элементы будут отключены.
Если нажата диалоговая кнопка OK, функция повторного посещения будет включена для выбранного сайта, и значок «календарь», добавленный к узлу сайта в дереве истории.

Отключить повторное посещение

Если выбрано, функция повторного посещения будет отключена для выбранного сайта.

API

Надстройка поддерживает следующие конечные точки API:

revisit /view /revisitList Возвращает список сайтов, разрешенных для повторного посещения
revisit / action / revisitSiteOn Включает повторное посещение для указанного сайта. Даты должны быть в формате: "гггг/мм/дд чч:мм:сс"
revisit / action / revisitSiteOff Отключает повторное посещение для указанного сайта.
