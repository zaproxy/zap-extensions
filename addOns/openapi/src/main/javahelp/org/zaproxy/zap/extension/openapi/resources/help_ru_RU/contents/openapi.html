Поддержка OpenAPI

Это дополнение позволяет просматривать и импортировать определения OpenAPI (Swagger) версий 1.2, 2.0 и 3.0.
Примечание: Генерация содержимого XML в настоящее время не поддерживается.

Надстройка автоматически обнаружит любые определения OpenAPI и просканирует их, пока они находятся в области действия.

Он также поддерживает Платформа автоматизации .

Интерфейс

В меню Импорт добавлены 2 пункта меню:
  • Импорт определения OpenAPI из локальной файловой системы
  • Импорт определения OpenAPI из URL-адреса
Both dialogues allow to override the server URL present in the OpenAPI definition (or specify one if not present) through the Target URL field. The import progress is shown in the progress tab.

Формат целевого URL

Целевой URL имеет следующий формат:
scheme://authority/path
со всеми необязательными компонентами URI. Примечание. Хотя все компоненты URI являются необязательными, схема и полномочия становятся обязательными, если они не импортируются из URL-адреса и в определении они не указаны (например, в нем нет серверов или схем).
Следуя некоторым примерам, переопределение:
  • Полномочия и путь: localhost/api/
  • Просто схема: https://
  • Просто полномочия: qa.example.com:9090
  • Просто путь: /dev/v3/

API

В API добавлены следующие операции:
  • ДЕЙСТВИЕ importFile (файл, цель)
  • ДЕЙСТВИЕ import Url (url, host Override)
Оба Цель и hostOverride поддержать Целевой URL формат, описанный ранее. Определения будут импортированы синхронно, и будут возвращены все предупреждения.

Узлы, управляемые данными

Когда схема OpenAPI содержит параметры пути, подключаемый модуль автоматически создает управляемые данными узлы либо в контексте по умолчанию, либо для контекста из предоставленного contextId. Например, следующее определение OpenAPI приведет как минимум к одному узлу, управляемому данными. openapi.yml ... /users/v1/{username}/email: ... parameters: - name: username in: path description: username to update email required: true schema: type: string ... Default Context > Structure > Structural Modifiers DDN0: (/)(.+?)(/.*) DDN1: (/users/v1/)(.+?)(/.*)

командная строка

Добавлены следующие параметры командной строки:
  • -openapifile <filename>: импортирует определение OpenAPI из файла с указанным именем.
  • -openapiurl <url>: импортирует определение OpenAPI с указанного URL-адреса.
  • -openapitargeturl <url> : целевой URL-адрес для переопределения URL-адреса сервера, присутствующего в определении OpenAPI.
Определения будут импортированы синхронно, а все предупреждения будут отображаться в командной строке.

Определения будут импортированы синхронно, а все исключены в командной строке.

Значения по умолчанию используются при импорте определений OpenAPI.
Их можно переопределить с помощью надстройки Form Handler, которая позволяет вам указывать свои собственные значения.
В большинстве случаев это будут простые значения (например, строки и целые числа), но в некоторых случаях вам может потребоваться указать структурированные значения, например 
{ "id": 0, "name": "Freda" }

Статистика

Надстройка ведет следующую статистику:
  • openapi.urls.added : общее количество URL-адресов, добавленных при импорте определений OpenAPI.
