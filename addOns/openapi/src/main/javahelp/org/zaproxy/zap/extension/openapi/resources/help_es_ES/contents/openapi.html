Soporte de especificación de Open API

Este complemento permite rastrear e importar definiciones (versiones 1.2 y 2.0) OpenAPI (Swagger).

El complemento detecta automáticamente cualquier definición Open API y las rastrea siempre que estén en el ámbito de la aplicación.

Interfaz de Usuario

Se agregan 2 elementos de menú en el menú Herramientas:
  • Importar una definición Open API desde el sistema archivo local
  • Importar una definición Open API desde un URL
El cuadro de diálogo de importar URL soporta un campo adicional opcional que permite anular el Host (y puerto) en la definición con una alternativa que se proporcione.
Esta opción no se proporciona para la importación de archivos ya que es fácil cambiar un archivo local para utilizar un host diferente.

API

Las siguientes operaciones se agregan a la API:
  • ACTION importFile (file)
  • ACTION importUrl (url, hostOverride)
Las definiciones se importarán sincrónicamente y se devolverá cualquier advertencia.

Línea de comandos

Se agregan las siguientes opciones de línea de comandos:
  • -openapifile <filename> : importa una definición Open API desde el nombre de archivo especificado
  • -openapiurl <url> : importa una definición Open API desde un URL especificado
Las definiciones se importarán sincrónicamente y cualquier advertencia se mostrará en la línea de comandos.

Valores especificados por el usuario

Se utilizan valores por defecto cuando se importan definiciones de Open API.
Estos pueden anularse utilizando el complemento Form Handler que permite especificar valores propios.
En la mayoría de los casos estos serán valores simples (como cadenas y enteros) pero en algunos casos podría necesitarse especificar valores estructurados, por ejemplo. 
{ "id": 0, "name": "Freda" }
