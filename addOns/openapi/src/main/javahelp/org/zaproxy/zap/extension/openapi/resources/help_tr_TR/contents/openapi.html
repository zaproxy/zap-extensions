Open API Specification Desteği

Bu eklenti OpenAPI (Swagger) tanımlamalarını (1.2 ve 2.0 versiyonları) spider'lamanızı ve içe aktarmanızı sağlar.

Bu eklenti, kapsamda oldukları sürece otomatik olarak herhangi bir Open API tanımlamasını algılayacak ve spider'layacaktır.

Kullanıcı Arayüzü

Araçlar menüsüne 2 menü öğresi eklenir:
  • Yerel dosya sisteminden bir Open API tanımlaması içe aktar
  • Bir URL'den Open API tanımlaması içe aktar
URL İçe Aktarma iletişim kutusu, tanımdaki Ana Bilgisayarı (ve bağlantı noktasını) sizin sağladığınız alternatif bir sunucu ile geçersiz kılmanıza izin veren isteğe bağlı bir ek alanı destekler.
Yerel bir dosyayı farklı bir host kullanmak için değiştirmek kolay olduğundan bu seçenek dosya içe aktarımı için sağlanmaz.

API

API'ya aşağıdaki işlemler eklenir:
  • EYLEM importFile (file)
  • EYLEM importUrl (url, hostOverride)
Tanımlamalar eş zamanlı olarak içe aktarılacaktır ve tüm uyarılar döndürülecektir.

Komut Satırı

Aşağıdaki Komut Satırı seçenekleri eklenir:
  • -openapifile <filename> : Belirtilen dosya isminden bir Open API tanımlaması içe aktar
  • -openapiurl <url> : Belirtilen URL'den bir Open API tanımlaması içe aktar
Tanımlar eşzamanlı olarak içe aktarılacak ve uyarılar komut satırında görüntülenecektir.

Kullanıcı Belirli Değerler

Varsayılan değerler açık API tanımlarını içe aktarırken kullanılır.
Bu, kendi değerlerinizi belirtmenizi sağlayan Form Handler eklentisi kullanarak override edilebilir.
Çoğu durumda bunlar basit değerler (string ve tam sayılar gibi) olacaktır ama bazı durumlarda yapılandırılmış değerler belirtmeniz gerekebilir, örneğin 
{ "id": 0, "name": "Freda" }
