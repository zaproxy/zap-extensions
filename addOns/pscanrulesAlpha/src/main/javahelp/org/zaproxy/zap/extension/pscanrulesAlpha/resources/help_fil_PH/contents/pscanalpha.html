Image lokasyon eskaner

Mga passively eskan para sa lokaayon ng GPS pag lantad ng image sa panahon ng normal ang seguridad ng mga assessment sa mga website. Image lokasyon eskaner tumotulong sa sitwasyon kung saan maaaring magpaskil ng mga larawan sa propayl ang end user at posibleng ipamigay ang kanilang lokasyon sa bahay e.g. a dating site or children's chatroom. Ang isang whitepaper na ito ay matatagpuan sa http://veggiespam.com/ils/

Tandaan: ang pagkakasunod sunod ng plug-in na ito para magamit sa, ZAP kaylangan isaayos para makatangap at maproseso ang mga image. Upang gawin ito, pumunta sa opsiyon ng ZAP panel (Tools → Opsiyon), pumili ng display, at paganahin ang checkbox para sa "pagpaprases ng mga image sa HTTP requests/responses". Para sa pagdadagdag, ng mga image sa ZAP ay hindi masasala sa pamamagitan ng mga setting "Global Exclude URL" o ang sesiyon prapertis para sa "Exclude from Proxy".

Buksan na Redirect

Ang mga bukas na redirect ay isa mga top ten vulnerabilities ng OWASP 2010. Ang pagsusuring ito ay tumitingen sa user-supplied input ng mga query string parameter at post data para matukoy kung saan posibleng bukas ang mga redirect. Ang mga bukas na redirect ay mangyayari kapag ang isang aplikasyon ay pinahintulotan ang user-supplied input (e.g. http://nottrusted.com) na makuntrol ang isang offsite redirect. Ito ay pangkaraniwan isang tumpak na paraan para mahanap ang 301 o 302 mga redirect na maaaring pagsamantalahan ng mga spammer o ng pag-atake sa phishing

HTTP server ay tumugon sa header eskaner

Ang pagsusuring ito ay tumutugon sa mga header para sa presensiya ng isang server header na naglalaman ng mga detalye ng bersiyon. Sa mahinang Threshold ay mataas ang alerto batay sa presensiya ng header field kung nakita o hindi ang bersyon ng string.

Istiktong Siguridad ng transport sa header eskaner

Itong eskaner ay sinusuri ang mga tugon ng HTTPS para sa presensiya ng Striktong Seguridad ng Transport HTTP (HSTS) header at pagsubok sa iba't ibang suliranin, Maging alerto kung natagpuan na ito. Mga alertong generated:
  • Strict-Transport-Security Header Not Set: kung ang tugon ay HTTPS at ang header ay ganap na nawawala.
  • Strikto-Transport-Seguridad Disabled: kung ang tugon ay HTTPS at ang halaga ng max-age ay naka-set sa zero, gayon pagnireset ang mga browser ang impormasiyon ng HSTS para sa site.
  • Strikto-Transport-Security maraming entre ang header (Hindi sumusunod sa Spec): kung ang tugon ay HTTPS at meron higit sa isang HSTS header sa mga tumugon.
  • Stricto-Transport-Seguridad Nawawalang Max-Age (hindi-sumusunod sa Spec): kung ang tugon ay HTTPS, isang HSTS header ay tiyak pero ang max-age value ay hindi naglalaman ng numero.
  • Strikto-Transport-Segutidad ay tinukoy sa pamamagitan ng META (Hindi sumosunod sa Spec): kung ang response body ay kabilang sa META tag sino ang susubok tukuyin ang HSTS.
  • Strikto-Transport-Seguridad ang Header ay nasa plain sa tugon ng HTTP: Kung ang tugon ay HTTP at ang HSTS header ay kasalukuyan.
  • Strikto-Transport-Seguridad Max-Age malformed (Hindi sumusunod sa Spec) kung ang tugon ay HTTPS at ang HSTS header ay kasalukuyan, ngunit meron mga quote na nauuna ang max-age directiba(ang mga quote ay pinapayagan sa paligid ng max-age value, ngunit hindi sa paligid ng directibang ito).
  • Stikto-Transport-Seguridad Malformed Nilalaman(hindi sumusunod sa Spec): kung ang tugon ay HTTPS at ang HSTS header ay kasalukuyan, ngunit meron mga hindi inaasahang nilalaman(tulad ng mga buhol buhol na quote). Ang inaasahan ay maipi-print ang nilalaman ng ASCII.

Ang gumagamit ay nakukuntrol ang katangian ng elemento sa HTML (potensyal XSS)

Ito ay nagbibigay ng pangtukoy sa hot-spot para sa XSS (cross-site scripting) na kakailanganin ng karagdagang pagsusuri ng isang analyst sa seguridad upang matukoy ang pagsasamantala.

Ang Gumagamit ay Nakukuntrol ang Charset

Ang pagsusuring ito ay tumitingen sa user-suplied input sa mga query string parameter at POST data upang makilala kung saan ang uri ng nilalaman o meta tag charser ang mga deklarasyon ay maaaring kontrolado ng gumagamit. Ang naturang mga deklarasyon ng charset ay dapat palaging ideklara ng aplikasyon. Kung ang umatake ay kayang kontrolin ang tugon ng charset maaari niyang manipulahin ang HTML para mag perporm sa XSS o iba pang mga pag atake.

User Kontrolable Javascript Event (XSS)

Ang pag susuring ito ay tumitingen sa user-supplied input ng query string parameters at post data para matukoy kung saan tiyak ang halaga ng HTML attribute na maaaring kontrolin. Ito ay nagbibigay ng pangtukoy sa hot-spot para sa XSS (cross-site scripting) na kakailanganin ng karagdagang pagsusuri ng isang analyst sa seguridad upang matukoy ang pagsasamantala.

Ang gumagamit ay nakukuntrol ang pag aari ng Javasript (XSS)

Ang pagsusuring ito ay tumitingen sa user-supplied input ng query string parameters at post data para makilala kung saan ang URL's ay tiyak ang mga katangian ng javascript (e.g.createElement src) maaaring becontrolled. Ito ay nagbibigay pangtukoy sa hot-spot para sa XSS (cross-site scripting) na kakailanganin ng karagdagan pagsusuri ng isang manunuri sa siguridad upang matukoy ang pagsasamantala.

Ang Username ng Hash ay natagpuan

Kung anuman ang nilalaman ng kongteksto na tinutukoy ng mga gumagamit itong iskaner ay sinusuri ang lahat ng mga tugon para sa presensiya ng hashed values na kumakatawan sa mga username. Pagkatuklas ng anumang naturang halaga ay maaaring kumakatawan sa isang Insecure Direct Object Reference (IDOR) na kahinaan. Ang alerto ay itataas lamang bilang inpormasyon aytem bilang further ang manu-manong pagsusuri ay kinakaylangan upang makompirma at masuri ang impact.
