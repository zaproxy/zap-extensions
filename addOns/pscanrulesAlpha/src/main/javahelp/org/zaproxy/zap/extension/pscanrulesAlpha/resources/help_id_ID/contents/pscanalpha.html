Secara sederhana tentu saja ini mengatur pemeriksaan kedekatan header "substansi-kebijakan-keamanan". Terlebih lagi, pada batas rendah juga memeriksa header "X-Content-Security-Policy" dan "X-WebKit-CSP".

Pemindai Lokasi Gambar

Secara pasif memindai pemajanan lokasi GPS dalam gambar selama penilaian keamanan normal berbagai laman. Pemindai Lokasi Gambar membantu dalam situasi di mana pengguna dapat memasang foto profil dan memberikan lokasi rumah mereka, misalnya situs kencan atau ruang chat anak-anak. Buku Putih tentang hal ini dapat ditemukan di http://veggiespam.com/ils/

Catatan: Agar plug-in ini dapat beroperasi, ZAP harus dikonfigurasi untuk menerima dan memproses gambar. Untuk melakukannya, pergi ke panel Pilihan ZAP (Perkakas → Opsi), pilih Tampilkan, dan aktifkan kontak centang untuk "Proses gambar dalam permintaan/respon HTTP". Sebagai tambahan, gambar ZAP tidak dapat disaring melalui pengaturan "URL Peniadaan Global" atau Properti Sesi untuk "Tiadakan dari Proksi".

Buka pengalihan

Pengalihan terbuka adalah salah satu dari Sepuluh Teratas kerentanan OWASP 2010. Pemeriksaan ini mengamati masukan yang diberikan oleh pengguna dalam parameter string kueri dan MENGIRIM data untuk mengidentifikasi di mana pengalihan terbuka mungkin dilakukan. Pengalihan terbuka terjadi ketika suatu aplikasi mengizinkan masukan yang diberikan oleh pengguna (misalnya http://nottrusted.com) untuk mengendalikan pengalihan di luar situs. Pemindai Tajuk Keamanan Transportasi Ketat

Pemindai ini memeriksa respons HTTPS akan kehadiran tajuk Keamanan Transportasi Ketat HTTP (HSTS) dan melakukan tes terhadap berbagai masalah implementasi, memberikan peringatan jika ditemukan. Peringatan-peringatan yang diberikan:
  • Strict-Transport-Security Header Not Set: Jika respons berupa HTTPS dan tajuknya hilang sama sekali.
  • Strict-Transport-Security Disabled: Jika respons berupa HTTPS dan nilai maksimal umur diatur ke nol, maka informasi HSTS peramban untuk situs akan ditata ulang.
  • Strict-Transport-Security Multiple Header Entries (Non-compliant with Spec): Jika respons berupa HTTPS dan ada lebih dari satu tajuk HSTS pada respons.
  • Strict-Transport-Security Missing Max-Age (Non-compliant with Spec): Jika respons berupa HTTPS, sebuah tajuk HTST ditentukan tapi nilai maks-umur tidak mengandung angka.
  • Strict-Transport-Security Defined via META (Non-compliant with Spec): Jika tubuh respons termasuk tag META yang berusaha menentukan HSTS.
  • Strict-Transport-Security Header on Plain HTTP Response: Jika respons berupa HTTP dan ada tajuk HSTS.
  • Strict-Transport-Security Max-Age Malformed (Non-compliant with Spec): Jika respons berupa HTTPS dan ada tajuk HSTS, tapi ada tanda kutip sebelum petunjuk umur maksimal (tanda kutip diperbolehkan di sekitar nilai maksimal, tapi tidak pada petunjuk itu sendiri).
  • Strict-Transport-Security Malformed Content (Non-compliant with Spec): Jika respons berupa HTTPS dan ada tajuk HSTS, tapi ada beberapa konten yang tak terduga (seperti tanda kutip kurawal). Harapannya adalah konten berupa ASCII yang bisa dicetak.
