Authentication status scanning

Having a good configuration does not necessarily mean authentication is successful. What if the credentials are wrong or provided logged in/out indicator is misleading. With this feature, you can check the authentication status of each URI. The scan results are shown in authentication status tab as shown in this picture


Authentication status scanning is started using authentication helper dialog.

Authentication status

Authentication status indicates whether ZAP can authenticate with the web application successfully or not. Unfortunately, for non-HTTP based authentication schemes, there is no perfect approach to determine authentication status programmatically. ZAP uses logged in indicator and logged out indicator to assess the authentication status. For each HTTP(S) request that is proxied through ZAP, the response message(both header and body) is scanned to match provided logged in and logged out indicators. Based on this result the authentication status is determined. As a result of this, the determined authentication status is just ZAP's understanding of authentication state, and it purely depends on the provided indicators.

As you already guessed, based on the presence of provided logged in indicator and logged out indicator in the response message, the authentication status can have four states. It must be noted that it is not necessary to define both indicators, however at least one of the indicator should be defined to start checking the authentication status.

Below table illustrates the four authentication states.


Authentication Status Icon Logged in indicator Logged out indicator
Successful authentication Found Not found or not defined
Failed authentication Not found or not defined Found
Conflicting authentication Found Found
Unknown authentication Not found Not found

Indicator status

As described earlier the authentication status is determined by the presence of logged in and logged out indicators in the response message. Below table describes the different state of an indicator.


Indicator Status Icon Description
Found The indicator is found in the response message
Not found The indicator is not found in the response message
Not defined User has not provided the indicator
Could not verify ZAP was unable to determine the indicator state
