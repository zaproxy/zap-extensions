Skrip

ZAP Script Add-on mengizinkan anda untuk menjalankan script yang bisa tertanam di dalam ZAP dan bisa mengakses internal ZAP struktur data.
Itu mendukung bahasa scripting apapun yang mendukung JSR 223 (http://www.jcp.org/en/jsr/detail?id=223) , termasuk:

  • ECMAScript / Javascript (menggunakan Nashorn mesin, dimasukkan secara default)
  • Zest https://developer.mozilla.org/en-US/docs/zest (dimuat secara bawaan)
  • Groovy http://groovy-lang.org/
  • Python http://www.jython.org
  • Ruby - http://jruby.org/
  • dan masih banyak lagi...
PERINGATAN - script dijalankan dengan perizinan yang sama seperti ZAP, sehingga tidak menjalankan script yang anda tidak percaya!

Jenis skrip

Berbagai jenis script yang didukung:
  • Berdiri Sendiri - script yang mandiri dan hanya berjalan ketika anda mulai mereka secara manual
  • Aktif Aturan ini dijalankan sebagai bagian dari aktiv scanner dan dapat diaktifkan secara individual
  • Pasif Aturan ini dijalankan sebagai bagian dari Pasif Scanner dan dapat diaktifkan secara individual
  • Aturan Proxy - ini menjalankan 'padagaris', bisa berubah pada setiap permintaan dan respon dan bisa dibolehkan secara individual. Mereka juga bisa memicu titik istirahat
  • HTTP Pengirim - script yang dijalankan terhadap setiap permintaan/respon yang dikirim/diterima oleh ZAP. Ini termasuk proksi pesan, pesan yang dikirim selama aktif scan, fuzzer, ...
  • Aturan Sasaran - script yang diminta dengan sebuah target URL dan hanya berjalan ketika anda memulainya secara manual
  • Otentikasi - script yang diminta ketika otentikasi yang dilaksanakan untuk sebuah Konteks. Untuk bisa digunakan, yang mereka butuhkan untuk bisa dipilih ketika mengkonfigurasi Script-Berdasarkan Metode Otentikasi untuk sebuah Konteks.
  • Script Input Vektor - script untuk mendefinisikan persis apa ZAP harus menyerang
  • Extenders - skrip yang bisa menambahkan fungsionalitas baru, termasuk didalamnya elemen grafis dan poin akhir API baru
Semua skrip yang dijalankan secara otomatis pada awalnya 'dinonaktifkan' - anda harus mengaktifkan hal tersebut melalui Script 'pohon' tab sebelum mereka akan berjalan.
Jika terjadi kesalahan ketika mereka berjalan, maka mereka akan dinonaktifkan.
Ketika anda memilih scrip kemudian yang terakhir kesalahan akan ditampilkan dalam Scrip konsol tab.
Skrip yang ditargetkan bisa dipanggil melalui klik kanan pada rekaman di tab Situs atau Riwayat dan memilih item menu 'Invoke with script...'.

Semua bahasa skrip bisa digunakan untuk semua jenis skrip, tetapi hanya bahasa yang telah diunduh dariMarketplace ZAP yang lazimnya akan memiliki template. Namun mungkin anda bisa menyesuaikan sebuah template untuk bahasa lain.
Jika bahasa favorit anda tidak tersedia pada Marketplace maka tolong sampaikan masalah baru melalui "Online / Report an issue" pada item menu.
Sementara itu anda hanya dapat meletakkan stoples yang relevan pada direktori 'lib' (bukan direktori 'plugin') dan mulai ulang ZAP.

Variabel Global

Variabel bisa dibagi diantara semua skrip melalui kelas org.zaproxy.zap.extension.script.ScriptVars.
Misalnya dalam Javascript anda dapat menggunakan kelas ini sebagai berikut:

org.zaproxy.zap.extension.script.ScriptVars.setGlobalVar("var.name","value")
org.zaproxy.zap.extension.script.ScriptVars.getGlobalVar("var.name")

Scrip Variabel

Variabel bisa dibagikan diantara permohonan terpisah dari script yang sama melalui yang sama org.zaproxy.zap.ekstensi.script.ScriptVars kelas.
Misalnya dalam Javascript anda dapat menggunakan kelas ini sebagai berikut:

org.zaproxy.zap.ekstensi.script.ScriptVars.setScriptVar(ini.konteks, "var.nama","nilai")
org.zaproxy.zap.ekstensi.script.ScriptVars.getScriptVar(ini.konteks, "var.nama")
Catatan bahwa metode ini hanya bisa digunakan dari bahasa skrip yang menyediakan akses ke ScriptContext (seperti Javascript).
